# 并行 Agent 测试方案

## 测试目标
验证 xuanji 的多 Agent 并行执行能力，包括：
- ✅ 并行调度正确性
- ✅ 独立任务互不阻塞
- ✅ 依赖关系正确处理
- ✅ 错误隔离（单个失败不影响其他）
- ✅ 资源竞争处理（文件读写）

---

## 测试用例 1：基础并行（独立任务）

### 场景描述
3 个完全独立的信息查询任务，无依赖关系。

### 测试命令
```
请帮我同时完成以下三个任务：
1. 查找项目中所有 TODO 注释
2. 统计 src/ 目录下 TypeScript 文件总数
3. 检查 package.json 中的脚本命令
```

### 预期行为
- 🔄 三个 SubAgent 同时启动
- 🎯 各自独立执行工具调用
- ⚡ 总耗时 ≈ max(单个任务耗时)，而非累加
- ✅ 结果汇总清晰展示

### 成功标志
- UI 显示 "⚙️ 并行执行 3 个任务"
- 日志显示 parallel=true
- 无 "waiting for xxx" 消息

---

## 测试用例 2：混合并行（串行依赖）

### 场景描述
- 任务 A、B 并行执行
- 任务 C 依赖 A 的结果

### 测试命令
```
请帮我完成：
1. 同时查找 "TODO" 和 "FIXME" 注释（可并行）
2. 统计第一步找到的总数，生成报告（依赖第一步）
```

### 预期行为
- 🔄 第一步：两个 grep 并行执行
- ⏸️ 第二步：等待第一步完成后执行
- 📊 报告包含两次查找的汇总

### 成功标志
- 第一步显示 2 个并行任务
- 第二步等待前置完成
- 依赖关系正确识别

---

## 测试用例 3：资源竞争（读写冲突）

### 场景描述
多个任务操作同一文件，测试冲突处理。

### 测试命令
```
创建测试文件 test-parallel.txt，内容为 "Initial"，然后同时：
1. 读取文件内容
2. 追加 "Line 1"
3. 追加 "Line 2"
```

### 预期行为
- ✅ 文件创建成功
- 🔄 读取操作并行执行
- 🔒 写入操作串行化（自动检测冲突）
- 📝 最终文件完整包含所有内容

### 成功标志
- 写入操作被串行化（或并发安全）
- 无文件损坏或丢失
- 错误处理清晰（如有）

---

## 测试用例 4：错误隔离

### 场景描述
并行任务中一个失败，其他继续执行。

### 测试命令
```
同时执行：
1. 读取存在的文件 package.json
2. 读取不存在的文件 /path/to/nonexist.txt
3. 统计 src/ 下文件数
```

### 预期行为
- ✅ 任务 1、3 成功完成
- ❌ 任务 2 报错但不影响其他
- 📊 结果分别展示成功和失败

### 成功标志
- 2 个任务成功，1 个失败
- 失败信息清晰展示
- 无整体任务中断

---

## 测试用例 5：压力测试（5+ 并行）

### 场景描述
大量独立任务同时执行，测试并发能力。

### 测试命令
```
请同时帮我：
1. 列出 src/core 下的文件
2. 列出 src/adapters 下的文件
3. 列出 src/mcp 下的文件
4. 统计 src/types 下的文件
5. 统计 src/learning 下的文件
6. 读取 README.md 的前 10 行
```

### 预期行为
- 🚀 6 个任务并行启动
- 📊 UI 清晰展示多任务状态
- ⚡ 整体耗时合理（< 15s）
- 🧠 内存占用稳定

### 成功标志
- 所有任务完成
- 无内存溢出
- 输出格式清晰可读

---

## 测试执行顺序

建议按以下顺序执行：
1. ✅ 测试 1（基础并行）
2. ✅ 测试 4（错误隔离）
3. ✅ 测试 2（混合依赖）
4. ✅ 测试 5（压力测试）
5. ⚠️ 测试 3（资源竞争，可能失败）

## 注意事项

1. **环境准备**
   - 确保重启后的 xuanji 版本包含修复
   - 清理之前的测试文件（test-parallel.txt）

2. **观察重点**
   - 日志中是否显示 `parallel=true`
   - UI 是否显示并行任务提示
   - 子任务输出是否清晰可区分

3. **问题排查**
   - 如果仍报错，检查 `DelegateTool` 修复是否生效
   - 如果串行执行，检查 `SubAgentCoordinator` 依赖分析
   - 如果 UI 混乱，检查 `renderSubAgents()` 实现
