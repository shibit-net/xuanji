# 并行工具 UI 测试说明

## 优化内容

### 1. 树状结构展示
- 使用 `┌─`, `├─`, `└─` 字符构建清晰的层级结构
- 并行工具组统一放在树状容器中
- 每个工具显示状态图标、名称、命令、进度/耗时

### 2. 动态执行区域
进行中的并行工具使用展开的树状结构:
```
┌─ ⚡ Parallel Execution (2/3 completed)
├─ ✓ Read file  src/index.ts  (0.12s)
├─ ⏳ Grep  pattern="foo" in /src  (1.2KB)
└─ ⏳ Glob  src/**/*.ts
```

### 3. 静态历史区域
完成的并行工具组:
- **折叠模式**: 单行紧凑显示 `⚡ Parallel (3 tools): Read file, Grep, Glob ✓ 0.15s`
- **展开模式**: 树状结构 + 边框，支持 Tab 导航和 Enter 切换

## 测试方法

运行以下命令触发并行工具执行（readonly 工具会自动并行）:

```bash
# 测试 1: 3个 read_file 并行
请同时读取 package.json、tsconfig.json、README.md 这三个文件

# 测试 2: grep 和 glob 并行
请搜索所有 TypeScript 文件中包含 "export" 的行，同时列出 src 目录下的所有 .tsx 文件

# 测试 3: 混合并行和串行
先读取 package.json，然后同时搜索 src 目录中的 import 语句和 export 语句
```

## 预期效果

### 动态区域 (执行中)
```
┌─ ⚡ Parallel Execution (1/3 completed)
├─ ✓ Read file  package.json  (0.08s)
├─ ⏳ Grep  pattern="export" in src  (2.5KB)
└─ ⏳ Glob  src/**/*.tsx
```

### 静态区域 (折叠)
```
⚡ Parallel (3 tools): Read file, Grep, Glob ✓ 0.23s
```

### 静态区域 (展开)
```
┌─────────────────────────────────────────────┐
│ ⚡ Parallel Execution (3 tools) · 0.23s ✓   │
│                                             │
│  ├─ ✓ Read file  package.json  (0.08s)     │
│  ├─ ✓ Grep  pattern="export" in src (0.12s)│
│  └─ ✓ Glob  src/**/*.tsx  (0.03s)          │

```

## 键盘交互
- **Tab**: 在工具结果之间导航 (包括并行组)
- **Enter**: 展开/折叠选中的工具或工具组
- **q**: 退出导航模式
