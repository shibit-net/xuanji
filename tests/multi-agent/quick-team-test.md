# QuickTeamTool 模板测试计划

## 测试目标
并行测试 QuickTeamTool 的三个核心模板，收集详细的执行数据和性能指标。

## 测试用例

### 测试 1: code-review 模板
**目标**: 审查 QuickTeamTool.ts 的代码质量、安全性和性能
**文件**: src/core/tools/QuickTeamTool.ts
**预期团队配置**:
- 策略: sequential (顺序执行)
- 成员: 架构师 → 安全专家 → 性能专家
- 预期执行顺序: 每个专家独立审查，输出独立报告

**评估标准**:
1. 团队是否按顺序执行
2. 每个成员是否给出专业建议
3. 是否发现真实的代码问题
4. 输出是否结构清晰

### 测试 2: research 模板
**目标**: 调研 TypeScript Agent 架构的最佳实践
**主题**: "TypeScript multi-agent system architecture patterns"
**预期团队配置**:
- 策略: parallel (并行执行)
- 成员: 文档研究员、代码示例分析员、社区实践研究员
- 预期执行: 三个成员同时搜索不同来源

**评估标准**:
1. 三个成员是否并行执行
2. 是否覆盖文档、代码、社区三个维度
3. 信息是否互补
4. 最终报告是否综合全面

### 测试 3: architecture-debate 模板
**目标**: 讨论团队协作的内存共享策略
**主题**: "Should team members share memory context during execution?"
**预期团队配置**:
- 策略: debate (辩论模式)
- 成员: 简洁派、可扩展派、实用派
- 预期执行: 3轮辩论达成共识

**评估标准**:
1. 是否有多轮辩论
2. 三个视角是否有差异
3. 是否逐步收敛到共识
4. 最终决策是否合理

## 性能指标收集

每个测试收集以下数据：
- 总执行时间
- 各成员执行时间
- Token 使用量（输入/输出）
- 执行轮数
- 是否超时
- 成功/失败状态
