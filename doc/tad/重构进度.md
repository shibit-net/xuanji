## 关键设计决策

参考设计文档：architecture-blueprint.md 第 2 节（核心状态机设计）、第 5 节（统一生命周期模型）

1. **ConversationStore 命名**：现有 `sessionStore.ts` 管理权限/日志/planMode，新 Store 管理会话状态。macOS 大小写不敏感，`SessionStore.ts` 与 `sessionStore.ts` 冲突，故命名 `ConversationStore.ts`。Phase 4 时合并到 sessionStore。

2. **AgentStateMachine 中的 import 路径**：使用相对路径 `'../utils/StoreEventBus'` 而非 `@/` 别名（`@/` → `src/`，不覆盖 `desktop/`）。

3. **EventForwarder 的 IPC sender 注入**：通过构造函数接收 `IpcSender` 回调，解耦自具体 IPC 机制，便于测试。

4. **SessionStateMachine 实现 InterruptChecker**：`SessionStateMachine` 实现了 `InterruptChecker` 接口，`shouldAbort()` 返回 `abortRequested`，`shouldStop()` 返回 `abortRequested || (pendingMessages.length > 0 && !isTextOutputStarted)`。Phase 2 时直接注入 AgentLoop，无需额外适配层。

5. **EventForwarder 映射表结构**：21 个事件声明式映射，分为 AgentEvents / HookEvents / AsyncTaskEvents 三组，每组映射函数统一管理（`mapSession` / `mapAgent` / `mapThinking` / `mapTask` / `mapTeam` / `mapTeamMember`），集中后约 10 行声明即可完成全部映射。

6. **AsyncTaskStateMachine 事件复用 EventBus**：不创建独立 IPC 通道，通过 `onTaskStateChanged` 回调 → EventForwarder → `agent:async-task-update` IPC 事件 → 前端 EventAdapter 消费。与 `HOOK_SUBAGENT_*` 事件共存，不替代。

7. **AgentStateMachine 的 ensureAgent 机制**：`transition()` 入口统一检查：如果 agent 不存在且事件不是 `AGENT_CREATED`，自动调用 `ensureAgent` 创建。消除原来 6 处分散的 `_promoteSubAgent` 调用。同时内置终态屏障：终态 agent 的非清理事件直接静默忽略。

8. **flowBuilder 的 dagre 布局**：使用 dagre 计算节点层级位置（`rankdir=TB`），team 节点包裹成员节点，与当前 React Flow 的 dagre 集成方式一致。`calculatePosition` 内部调用 `dagre.layout(g)`。

4. **幂等性**：`registerEventAdapter()` 用模块级 `let registered = false` 防止 React Strict Mode 双挂载。
