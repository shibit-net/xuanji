# Xuanji 记忆系统机制详解

## 当前记忆触发和使用机制

### 1. 记忆写入（Memory Write）

#### 触发时机
- **会话结束时** - 主要触发点
- **用户显式命令** - `/memory add`, `/identity set-name` 等

#### 执行 Agent
**MemoryFlushAgent**（记忆刷新 Agent）

**位置**：`src/memory/MemoryFlushAgent.ts`

**调用链**：
```
ChatSession.legacy.ts (会话结束)
    ↓
MemoryService.flushOnExit()
    ↓
MemoryFlushAgent.flushOnExit()
    ↓
SubAgentFactory.createSubAgent('memory-extractor')
    ↓
memory-extractor SubAgent 分析对话
    ↓
MemoryManager.save() 存储到 memory.db
```

**工作流程**：
1. 会话结束时，ChatSession 调用 `memoryService.flushOnExit(messages)`
2. MemoryService 调用 `memoryFlushAgent.flushOnExit(messages)`
3. MemoryFlushAgent 创建 SubAgent（memory-extractor）
4. SubAgent 分析整个对话历史，提取：
   - 值得记忆的信息
   - 经验教训（错误、改进、最佳实践）
   - 成功模式（可复用经验）
   - 未完成任务
5. 将提取的记忆存储到 memory.db

**特点**：
- ✅ 异步执行，不阻塞主流程
- ✅ 使用专门的 SubAgent 分析
- ✅ 超时保护（30 秒）
- ✅ 自动分类（scope, volatility, significance）

### 2. 记忆读取（Memory Read）

#### 触发时机
- **每次主 Agent 响应前** - 构建 System Prompt 时

#### 执行 Agent
**主 Agent**（通过 PromptOrchestrator）

**位置**：`src/core/chat/PromptOrchestrator.ts`

**调用链**：
```
主 Agent 收到用户输入
    ↓
PromptOrchestrator.buildAndApply()
    ↓
MemoryManager.formatDecisionContext(query)
    ↓
MemoryRetriever.retrieve(query) - 检索相关记忆
    ↓
IdentityManager.getIdentity() - 获取身份信息
    ↓
CoreRuleStore.getActiveRules() - 获取核心规则
    ↓
MemoryFormatter.format() - 格式化为 Prompt
    ↓
注入到 System Prompt
```

**工作流程**：
1. 主 Agent 收到用户输入
2. PromptOrchestrator 调用 `memoryManager.formatDecisionContext(query)`
3. MemoryManager 执行：
   - **检索相关记忆**：使用向量检索 + FTS 全文检索
   - **获取身份信息**：用户称呼、助手名字
   - **获取核心规则**：永久约束
4. MemoryFormatter 格式化为结构化文本
5. 注入到 System Prompt 的最前面

**特点**：
- ✅ 同步执行，确保记忆可用
- ✅ 智能检索，只返回相关记忆
- ✅ 分层注入（核心规则 > 身份 > 相关记忆）
- ✅ 权重计算（时间衰减 + 访问加成）

## 记忆存储位置

### memory.db (SQLite)

**存储内容**：
- 所有记忆条目（MemoryEntry）
- 核心规则（通过 PermanentConstraintManager）
- 身份信息（通过 IdentityManager）
- 向量索引（sqlite-vec）

**字段标记**：
```typescript
// 永久约束
{
  constraint: 'must',
  volatility: 'permanent',
  scope: 'core_rule' | 'profile',
}

// 普通记忆
{
  constraint: 'may' | 'should',
  volatility: 'stable' | 'normal' | 'transient',
  scope: 'knowledge' | 'episode',
}
```

## 当前问题和改进方向

### 问题 1：记忆写入时机单一
**现状**：只在会话结束时写入记忆

**问题**：
- 用户说"记住，我希望被称呼为 Boss"时，需要等到会话结束才会记忆
- 如果会话异常退出，记忆会丢失

**改进方向**：
- 检测用户显式记忆意图（"记住"、"不要忘记"）
- 立即调用记忆 Agent 异步存储
- 主 Agent 立即响应"好的，我会记住的"

### 问题 2：记忆分析在主流程中
**现状**：MemoryFlushAgent 在会话结束时同步分析

**问题**：
- 会话结束时需要等待记忆分析完成
- 增加了退出延迟

**改进方向**：
- 完全异步化，不阻塞会话退出
- 后台队列处理

### 问题 3：智能分类未集成
**现状**：已实现 ConstraintClassifier 和 SmartMemoryStorage，但未集成到主流程

**问题**：
- 无法自动判断是永久约束还是普通记忆
- 需要用户使用特定命令

**改进方向**：
- 在 MemoryFlushAgent 中集成智能分类
- 自动识别并正确存储

## 理想的记忆架构（下一步）

### 记忆 Agent 专职负责

```
用户输入："记住，我希望被称呼为 Boss"
    ↓
主 Agent 检测记忆意图
    ↓
立即响应："好的，我会记住的"
    ↓
异步调用记忆 Agent
    ↓
记忆 Agent 分析：
  - 这是永久约束（identity）
  - 需要更新旧的用户称呼
    ↓
存储到 memory.db
    ↓
下次对话时自动使用
```

**优点**：
- 主 Agent 快速响应
- 记忆分析专业化
- 用户体验流畅
- 架构清晰
