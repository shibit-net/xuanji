# GUI 拟人化交互设计文档（面向用户）

## 设计理念

Xuanji 是**用户的 AI 助手**，不是面向某个具体项目，而是**面向用户的所有诉求**：
- 💬 **日常对话**：问答、聊天、建议
- 📝 **写作助手**：文章、文档、邮件
- 📚 **学习伴侣**：知识查询、概念解释
- 💻 **编程帮手**：代码、调试、重构
- 🎯 **生活助理**：提醒、规划、建议

创建一个**优雅、简洁但美观**的交互界面，通过**拟人化设计**让用户直观感受 Agent 的工作过程。

## 核心创意

### 左侧 - 当前关注面板（Agent 在看什么）
展示 Agent 当前关注的内容，让决策过程更透明：
- 📄 **文件**：Agent 访问过的文件（代码、文档、图片等）
- 💾 **记忆**：Agent 记住的关于你的信息
- ⏱️ **活动**：最近的对话和操作时间线

> **设计原则**：不假设用户是开发者，不假设用户在做"项目"

### 右侧 - Agent 拟人化面板（Agent 在做什么）
**核心创新**：以拟人化方式展示 Agent 的工作状态

## 拟人化设计元素

### 1. Agent 头像系统
每个 Agent 都有独特的视觉标识：

| Agent 类型 | 头像 | 颜色渐变 | 职责 |
|-----------|------|---------|------|
| **主 Agent 璇玑** | ⭐ | 紫→粉 (purple-500 to pink-500) | 主对话、任务分析 |
| **Worker Agent** | 🤖 | 蓝→绿→橙 | 执行子任务 |
| **Planner Agent** | 🧠 | 金色 (yellow-500 to amber-500) | 任务规划 |

### 2. 状态指示器
实时展示 Agent 当前状态（带动画）：

| 状态 | 图标 | 颜色 | 动画 |
|------|------|------|------|
| **空闲** | 😌 | gray-400 | 无 |
| **思考中** | 💭 | blue-500 | 脉冲 (1.5s 循环) |
| **执行中** | ⚙️ | green-500 | 缩放动画 |
| **等待** | ⏸️ | yellow-500 | 无 |
| **完成** | ✅ | green-600 | 无 |
| **错误** | ❌ | red-500 | 无 |

### 3. 思考气泡
模拟人类思考，让 Agent 更有温度：

```
┌─────────────────────────┐
│ 💭 正在思考...         │  ← 渐变边框 + 脉冲动画
│ 我来帮你查一下...       │
└─────────────────────────┘
      ↑
   气泡尾巴（CSS transform rotate）
```

## 功能模块

### 当前关注面板（ContextPanel）

**3 个标签**：

#### 📄 文件
- 展示 Agent 最近访问的文件
- 支持各种文件类型：代码、文档、图片等
- 显示操作类型（读取、写入、编辑）和时间

#### 💾 记忆
- Agent 记住的关于用户的信息
- 类型：偏好 💝、知识 📚、决策 💡、经验 ⚡、问题 🔧
- 支持搜索和相似度匹配

#### ⏱️ 活动
- 最近的对话和操作时间线
- 用户提问、Agent 操作
- 相对时间显示（刚刚、5分钟前）

### Agent 拟人化面板（AgentPanel）

展示 Agent 的"生命"：
- 🎭 **拟人化头像**：渐变色 + 状态指示器
- 💭 **思考气泡**：Agent 的"内心活动"
- 📖 **工具卡片**：当前正在使用的工具
- ⏱️ **操作历史**：最近 8 条工具调用记录
- 📊 **统计面板**：总调用、成功、错误数

```
└─ 🩺 系统诊断
```

## 使用场景示例

### 场景 1：学生写论文
```
用户："帮我写一篇关于气候变化的论文大纲"

当前关注面板：
  📄 文件：climate-research.pdf (读取)
  💾 记忆：用户是环境科学专业学生
  ⏱️ 活动：查询气候变化资料

Agent 面板：
  ⭐ 璇玑 - 思考中
  💭 我先整理一下论文结构...
  🌐 正在搜索最新气候数据
```

### 场景 2：上班族管理任务
```
用户："提醒我明天下午2点开会"

当前关注面板：
  💾 记忆：用户工作日程偏好
  ⏱️ 活动：设置提醒

Agent 面板：
  ⭐ 璇玑 - 执行中
  ⚙️ 创建提醒任务
  ✅ 已设置提醒
```

### 场景 3：开发者编程
```
用户："重构这个函数，让它更简洁"

当前关注面板：
  📄 文件：utils.ts (读取 → 编辑)
  💾 记忆：用户喜欢函数式编程风格
  ⏱️ 活动：代码重构

Agent 面板：
  ⭐ 璇玑 - 执行中
  📖 read_file utils.ts
  ✏️ edit_file utils.ts
```

## 设计原则总结

1. ✅ **面向用户**：不假设用户是开发者或在做"项目"
2. ✅ **场景通用**：支持学习、写作、编程、生活等各种场景
3. ✅ **优雅简洁**：信息层次清晰，不冗余
4. ✅ **视觉美观**：渐变色、圆角、阴影、动画
5. ✅ **拟人化**：头像、表情、思考气泡、状态
6. ✅ **信息透明**：左侧看输入（关注），右侧看输出（行动）
7. ✅ **功能解耦**：会话管理、配置、状态展示完全独立

## 文案指南

### ✅ 用户友好的文案
- "当前关注"（不是"上下文"）
- "Agent 记住的关于你的信息"（不是"项目知识"）
- "最近的对话和操作"（不是"工具调用历史"）
- "读取文件"（不是"read_file"）

### ❌ 避免技术化表述
- ~~"项目信息"~~
- ~~"代码质量分析"~~
- ~~"工具调用"~~
- ~~"API 接口"~~
