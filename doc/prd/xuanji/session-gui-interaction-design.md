# 记忆驱动会话管理 GUI 交互设计

## 设计日期
2026-03-16

## 概述

基于"记忆驱动会话管理系统"的核心理念，设计一套全新的 GUI 交互模式，告别传统的"时间倒序消息列表"，拥抱"语义组织 + 智能检索 + 上下文恢复"的交互范式。

---

## 一、整体布局设计

### 1.1 主界面布局（三栏）

```
┌────────────────────────────────────────────────────────────────┐
│  璇玑 Xuanji                                    [设置] [帮助]   │
├─────────────┬──────────────────────────┬───────────────────────┤
│             │                          │                       │
│  会话导航    │      对话区 / 摘要区       │    上下文面板         │
│  (左侧栏)    │      (主区域)            │    (右侧栏)          │
│             │                          │                       │
│  - 智能分组  │  - 当前会话对话          │  - 会话摘要          │
│  - 语义搜索  │  - 或会话卡片网格        │  - 关键点            │
│  - 时间线    │  - 或会话详情            │  - 生成记忆          │
│  - 标签云    │                          │  - 相关会话          │
│             │                          │  - 记忆引用          │
│             │                          │                       │
│  220px      │      主区域               │     280px            │
│             │                          │                       │
└─────────────┴──────────────────────────┴───────────────────────┘
```

### 1.2 模式切换

支持三种视图模式：

1. **对话模式**（默认）：当前会话的对话区
2. **探索模式**：会话卡片网格 + 语义搜索
3. **图谱模式**：会话关联知识图谱

## 十、用户测试场景

### 场景 1: 新用户首次使用

**目标**: 验证新手引导和基础功能易用性

1. 用户启动应用 → 看到欢迎引导
2. 开始第一次对话 → 自然交互
3. 对话结束后提示保存 → 用户点击保存
4. 看到摘要确认弹窗 → 理解摘要和关键点概念
5. 保存成功 → 左侧出现会话卡片

**预期结果**:
- 用户能理解会话保存的价值
- 用户能看懂 AI 生成的摘要
- 用户知道如何恢复会话

### 场景 2: 老用户恢复历史会话

**目标**: 验证会话恢复的上下文加载效果

1. 用户打开应用 → 看到历史会话列表
2. 在左侧点击一个 2 天前的会话
3. 右侧显示预览（摘要、关键点、记忆）
4. 点击"恢复会话" → 加载动画
5. 对话区显示上下文横幅 + 最近 10 条消息
6. 右侧面板显示详细摘要和引用的记忆

**预期结果**:
- 用户能快速理解之前的对话内容（通过摘要）
- 用户看到相关记忆，感受到 AI "记住了"上下文
- 用户可以直接继续对话，无需重新说明背景

### 场景 3: 用户搜索相关会话

**目标**: 验证语义搜索的准确性和便利性

1. 用户想找之前关于"登录问题"的会话
2. 在左侧搜索框输入"修复登录"
3. 实时显示 3 个相关建议
4. 悬停第一个建议 → 快速预览弹窗
5. 点击查看详情 → 切换到探索模式
6. 看到按相似度排序的结果

**预期结果**:
- 用户能快速找到目标会话（无需记住精确名称）
- 相似度可视化帮助判断哪个会话最相关
- 快速预览避免打开错误会话

### 场景 4: 用户查看会话关联

**目标**: 验证知识图谱的可视化效果

1. 用户在某个会话详情页
2. 看到"相关会话 (2)"
3. 点击"全部" → 切换到图谱模式
4. 看到当前会话与其他会话的连线
5. 点击某个关联节点 → 跳转到该会话

**预期结果**:
- 用户能直观看到会话之间的关系（主题、记忆、项目）
- 用户发现之前没注意到的相关会话
- 用户理解会话不是孤立的，而是形成知识网络

   - 记忆引用实时展示

   - 对话模式：专注当前交互
   - 探索模式：浏览和搜索会话
   - 图谱模式：可视化知识关联
