# GUI 拟人化交互设计文档

## 设计理念

创建一个**优雅、简洁但美观**的 AI Agent 交互界面，通过**拟人化设计**让用户直观感受 Agent 的工作过程。

## 核心创意

### 左侧 - 上下文面板（Context Panel）
展示对话的背景信息，让 Agent 的判断更透明：
- 📂 **项目信息**：项目名称、路径、文件统计
- 📄 **文件访问记录**：最近读取/编辑的文件
- 💾 **记忆检索**：相关的历史记忆
- ⏱️ **最近活动**：用户输入和工具调用时间线

### 右侧 - Agent 拟人化面板（Agent Panel）
**核心创新**：以拟人化方式展示 Agent 的工作状态

## 拟人化设计元素

### 1. Agent 头像系统
每个 Agent 都有独特的视觉标识：

| Agent 类型 | 头像 | 颜色渐变 | 职责 |
|-----------|------|---------|------|
| **主 Agent 璇玑** | ⭐ | 紫→粉 (purple-500 to pink-500) | 主对话、任务分析 |
| **Worker Agent** | 🤖 | 蓝→绿→橙 | 执行子任务 |
| **Planner Agent** | 🧠 | 金色 (yellow-500 to amber-500) | 任务规划 |

### 2. 状态指示器
实时展示 Agent 当前状态（带动画）：

| 状态 | 图标 | 颜色 | 动画 |
|------|------|------|------|
| **空闲** | 😌 | gray-400 | 无 |
| **思考中** | 💭 | blue-500 | 脉冲 (1.5s 循环) |
| **执行中** | ⚙️ | green-500 | 缩放动画 |
| **等待** | ⏸️ | yellow-500 | 无 |
| **完成** | ✅ | green-600 | 无 |
| **错误** | ❌ | red-500 | 无 |

### 3. 思考气泡
模拟人类思考，让 Agent 更有温度：

```
┌─────────────────────────┐
│ 💭 正在思考...         │  ← 渐变边框 + 脉冲动画
│ 我需要先读取这个文件... │
└─────────────────────────┘
      ↑
   气泡尾巴（CSS transform rotate）
```

**特点**：
- 思考中：蓝紫渐变背景 + 边框 + 省略号动画
- 完成后：纯色背景，无动画

### 4. 工具调用卡片
可视化 Agent 的"行动"：

```
┌─────────────────────────┐
│ 📖 read_file    ⏳     │  ← 运行中：黄色边框
│ 42ms                    │
└─────────────────────────┘

┌─────────────────────────┐
│ 📝 write_file    ✅    │  ← 成功：绿色边框
│ 128ms                   │
└─────────────────────────┘

┌─────────────────────────┐
│ 💻 bash         ❌     │  ← 失败：红色边框
│ Error: command failed   │
└─────────────────────────┘
```

### 5. 执行时间线
展示 Agent 的工作历史（反向时序）：

```
12:34:56 ✓ read_file
12:34:55 ✗ bash
12:34:52 ✓ grep
12:34:50 ✓ glob
...
```

## 模式切换逻辑

### Chat 模式（对话）
```
显示：
  Sidebar (最左，独立存在)
  + ContextPanel (左，可切换)
  + ChatArea + InputArea (中)
  + AgentPanel (右，可切换)
```

### Settings 模式（设置）
```
显示：
  Sidebar (最左，独立存在)
  + SettingsPanel (全屏覆盖中间和右侧区域)

点击 SettingsPanel 的关闭按钮 → 返回 Chat 模式
```

### Agents 模式（Agent 配置）
```
显示：
  Sidebar (最左，独立存在)
  + AgentManager (全屏覆盖中间和右侧区域)

点击 AgentManager 的关闭按钮 → 返回 Chat 模式
```

## 技术实现

### 依赖
- **framer-motion**: 流畅的动画效果
- **lucide-react**: 图标系统
- **TailwindCSS**: 样式框架
- **zustand**: 状态管理

### 核心组件

#### ContextPanel.tsx
```typescript
- ProjectTab: 项目信息卡片
- FilesTab: 最近访问文件列表（从工具调用提取）
- MemoryTab: 记忆检索（集成 HybridRetriever）
- RecentTab: 活动时间线
```

#### AgentPanel.tsx
```typescript
- AgentCard: 拟人化 Agent 卡片
  - AgentAvatar: 渐变头像 + 状态指示器
  - StatusIndicator: 状态图标 + 动画
  - ThinkingBubble: 思考气泡
  - CurrentToolCard: 当前工具调用卡片
- ToolCallHistory: 工具调用历史时间线
- AgentStats: 底部统计面板
```

### 动画细节

#### 1. Agent 卡片进入/退出
```typescript
initial={{ opacity: 0, y: 20 }}
animate={{ opacity: 1, y: 0 }}
exit={{ opacity: 0, y: -20 }}
```

#### 2. 状态指示器脉冲
```typescript
animate={{ scale: [1, 1.2, 1] }}
transition={{ repeat: Infinity, duration: 1.5 }}
```

#### 3. 思考气泡省略号
```typescript
animate={{ opacity: [0, 1, 0] }}
transition={{ repeat: Infinity, duration: 1.5 }}
```

#### 4. 工具卡片滑入
```typescript

animate={{ opacity: 1, x: 0 }}
```

#### 5. 时间线项目渐显
```typescript
transition={{ delay: idx * 0.05 }}  // 逐项延迟
```

## 交互逻辑

### Agent 状态推断
从消息流实时推断 Agent 状态：

```typescript
// 思考中：assistant 消息但无内容
if (role === 'assistant' && !content) → status = 'thinking'

// 执行中：有未完成的工具调用
if (toolCalls.some(tc => tc.status === 'running')) → status = 'executing'

// 错误：工具调用失败
if (toolCalls.some(tc => tc.status === 'error')) → status = 'error'

// 完成：有内容 / 工具全部成功
else → status = 'done'
```

### 面板切换
通过标题栏"功能"菜单：
- **⌘ [** 切换上下文面板
- **⌘ ]** 切换 Agent 面板
- 独立于旧的工具面板（暂时保留）

### 状态同步
- 从 `useChatStore` 读取消息流
- 通过 `useEffect` 监听消息变化
- 自动更新 Agent 状态和工具调用历史

## 设计亮点

### 1. 拟人化温度
- 头像 + 表情让 Agent 更有个性
- 思考气泡模拟人类内心活动
- 状态动画带来"生命感"

### 2. 信息透明
- 左侧：用户知道 Agent 看到了什么（上下文）
- 右侧：用户知道 Agent 在做什么（行动）
- 中间：对话内容（结果）

### 3. 视觉层次
- 渐变色区分不同 Agent 角色
- 边框颜色表示工具调用状态
- 动画吸引注意力到活跃区域

### 4. 性能优化
- AnimatePresence 管理列表动画
- useMemo 缓存计算结果
- 限制时间线显示数量（最多 8 条）

## 设计原则总结

1. ✅ **优雅简洁**：信息层次清晰，不冗余
2. ✅ **视觉美观**：渐变色、圆角、阴影、动画
3. ✅ **拟人化**：头像、表情、思考气泡、状态
4. ✅ **信息透明**：上下文 + 行动 = 完整图景
5. ✅ **性能流畅**：动画 60fps，计算优化
6. ✅ **功能解耦**：会话管理和 Agent 配置完全独立

## 🔄 架构优化记录

### 问题：会话管理和 Agent 配置耦合（2026-03-14 修复）

**之前的错误设计**：
```typescript
// Sidebar 只在非 chat 模式显示
{viewMode !== 'chat' && sidebarVisible && <Sidebar />}
```

**问题**：
- 会话列表和 Agent 配置绑定在一起
- Chat 模式下看不到会话列表
- 用户想切换会话必须退出对话模式

**修复后的正确设计**：
```typescript
// Sidebar 始终独立存在（所有模式）
{sidebarVisible && <Sidebar />}

// ContextPanel 只在 Chat 模式显示
{viewMode === 'chat' && contextPanelVisible && <ContextPanel />}

// AgentManager 作为独立全屏页面
{viewMode === 'agents' && <AgentManager />}
```

**效果**：
| 组件 | 职责 | 独立性 |
|------|------|--------|
| **Sidebar** | 会话列表 + 新建会话 | ✅ 完全独立，所有模式都显示 |
| **AgentManager** | Agent 配置管理 | ✅ 完全独立，全屏页面 |
| **ContextPanel** | 项目上下文 | ✅ 独立，仅 Chat 模式显示 |
| **AgentPanel** | Agent 实时状态 | ✅ 独立，仅 Chat 模式显示 |
