## ✅ 已完成的改进

### **改进 1: Sequential 策略增强** ✅

**文件**: `src/core/agent/team/TeamManager.ts`

**改动**: 在 `enrichTaskForMember()` 方法中，当策略为 `sequential` 时，自动注入前置成员的所有结果。

**之前**：
```typescript
// Sequential 成员只能看到共享知识库，看不到前置成员的输出
```

**现在**：
```typescript
if (this.context!.config.strategy === 'sequential' && previousResults.length > 0) {
  const previousContext = previousResults
    .map((r, idx) => `[Step ${idx + 1} - ${r.memberId}]\n${r.result}`)
    .join('\n\n---\n\n');

  enriched = `Previous team members' work:\n\n${previousContext}\n\n${'='.repeat(50)}\n\nYour task (building on the above):\n${task}`;
}
```

**效果**：
- ✅ 每个成员都能看到前面所有成员的工作成果
- ✅ 支持渐进式任务完成（代码审查、多角度分析等）
- ✅ 保持原有 parallel / pipeline / hierarchical 策略不变

### **改进 2: 新增 ChainTool** ✅

**文件**: `src/core/tools/ChainTool.ts`

**功能**: 链式代理工具，允许 LLM 定义一个 Agent 链，自动串行执行。

**工具定义**：
```typescript
{
  name: 'agent_chain',
  description: 'Execute a chain of agents sequentially...',

  input_schema: {
    chain: [
      {
        agent_id: 'data-extractor',
        task_template: '提取数据',
      },
      {
        agent_id: 'data-cleaner',
        task_template: '清洗 {{previous_output}}',  // 自动替换
      },
      {
        agent_id: 'data-analyst',
        task_template: '分析 {{previous_output}}',
      }
    ]
  }
}
```

**特性**：
- ✅ 单次工具调用完成整个工作流（避免多次 ReAct 迭代）
- ✅ 自动上下文传递（`{{previous_output}}` 模板替换）
- ✅ 每步的输入、输出、耗时都记录
- ✅ 任一步失败，整个链中断并返回已完成的步骤信息
- ✅ 支持自定义 timeout（每步独立配置）

**优势对比**：

| 方式 | 迭代次数 | Token 消耗 | 错误处理 | 灵活性 |
|------|---------|----------|---------|--------|
| 手动编排（多次 task） | N+1 次 | 高 | LLM 决策 | ⭐⭐⭐⭐⭐ |
| **ChainTool** | **1 次** | **低** | **自动** | ⭐⭐⭐ |
| TeamTool pipeline | 1 次 | 低 | 自动 | ⭐⭐ |

### **改进 3: 新增 ListAgentsTool** ✅

**文件**: `src/core/tools/ListAgentsTool.ts`

**功能**: 列出所有可用的 Agent，帮助 LLM 做出更好的决策。

**工具定义**：
```typescript
{
  name: 'list_agents',
  description: 'List all available agents in the system',

  input_schema: {
    filter: {
      tags: ['finance', 'coding'],       // 标签过滤
      search: '股票',                     // 关键词搜索
      enabled_only: true,                 // 只显示启用的
      include_subagents: true             // 包含子代理
    }
  }
}
```

**返回信息**：
- Agent ID（用于 task / agent_chain）
- 名称和描述
- 能力列表
- 标签（领域）
- 模型配置
- 工具数量
- 是否为内置/自定义

**使用场景**：
```
LLM: "我需要分析股票数据，但不确定用哪个 Agent"
  → 调用 list_agents({ filter: { tags: ['finance'] } })
  → 系统返回：stock-analyst, investment-advisor, risk-assessor
  → LLM 基于描述选择 stock-analyst
  → 调用 task({ agent_id: 'stock-analyst', ... })
```

### **改进 4: 新增 MatchAgentTool** ✅

**文件**: `src/core/tools/MatchAgentTool.ts`

**功能**: 智能 Agent 匹配，根据任务描述自动推荐最佳 Agent。

**工具定义**：
```typescript
{
  name: 'match_agent',
  description: 'Find the best agent for a specific task...',

  input_schema: {
    task_description: '分析 $AAPL 的财报数据',
    domain_hint: 'finance',  // 可选
    top_k: 3                 // 返回前 3 个推荐
  }
}
```

**智能匹配算法**：

```
总分 = 向量相似度(40%) + 关键词匹配(30%) + 标签匹配(20%) + 能力匹配(10%)
```

**评分维度**：

1. **向量相似度**（40% 权重）
   - 任务描述 embedding vs Agent 能力描述 embedding
   - 余弦相似度计算
   - 需要 EmbeddingService

2. **关键词匹配**（30% 权重）
   - 提取任务关键词
   - 匹配 Agent 的 name / description / capabilities / tags

3. **标签匹配**（20% 权重）
   - domain_hint 与 Agent.tags 匹配
   - 二值匹配（有/无）

4. **能力匹配**（10% 权重）
   - 任务描述与 Agent.capabilities 重叠度

**返回示例**：
```
Top 3 recommended agents:

1. **股票分析师** (stock-analyst)
   Match: ★★★★★ 92%
   Reason: High semantic similarity, Strong keyword match, Domain match
   Breakdown: Semantic: 85%, Keyword: 60%, Domain: 100%, Capability: 70%

2. **投资顾问** (investment-advisor)
   Match: ★★★★☆ 78%
   Reason: Domain match, Relevant capabilities
   Breakdown: Semantic: 72%, Keyword: 45%, Domain: 100%, Capability: 60%

3. **风险评估** (risk-assessor)
   Match: ★★★☆☆ 65%
   Reason: Domain match
   Breakdown: Semantic: 60%, Keyword: 30%, Domain: 100%, Capability: 40%
```

## 🎯 决策机制

### **分层决策模型**

```
┌─────────────────────────────────┐
│   Level 1: 用户意图理解          │  主 Agent (LLM)
│   • 识别任务类型和领域            │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│   Level 2: 工具选择              │  主 Agent (LLM)
│   • task / chain / team         │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│   Level 3: Agent 选择            │  混合模式
│   • 直接指定                     │  - LLM 主导
│   • 系统推荐（list/match）       │  - 系统辅助
│   • 自动匹配（未来）             │
└─────────────────────────────────┘
              ↓
┌─────────────────────────────────┐
│   Level 4: 执行与监控            │  系统自动
│   • 上下文传递                   │
│   • 结果聚合                     │
└─────────────────────────────────┘
```

### **核心原则**

✅ **LLM 主导**: 所有关键决策由主 Agent (LLM) 做出
✅ **系统辅助**: 提供工具帮助 LLM 做出更好的决策
✅ **用户可干预**: 用户可以明确指定 Agent
✅ **渐进增强**: 从手动编排 → 辅助推荐 → 自动路由（未来）

## 📊 上下文传递机制对比

| 策略/工具 | 上下文传递方式 | 自动化程度 | 适用场景 |
|----------|--------------|----------|---------|
| **TeamTool - Pipeline** | output → input | ⭐⭐⭐⭐⭐ | 数据流水线 |
| **TeamTool - Sequential** | 前置结果注入 | ⭐⭐⭐⭐⭐ | 多角度分析 |
| **TeamTool - Hierarchical** | Leader → Workers | ⭐⭐⭐⭐ | 分层协作 |
| **ChainTool** | {{previous_output}} | ⭐⭐⭐⭐⭐ | 工作流编排 |
| **手动编排（task）** | LLM 手动传递 | ⭐ | 复杂决策 |

## 🚀 实际应用示例

### **示例 1: 数据处理工作流（ChainTool）**

```typescript
// LLM 决策使用 ChainTool
agent_chain({
  chain: [
    {
      agent_id: 'data-extractor',
      task_template: '从 stocks.csv 提取所有 $AAPL 的数据',
      description: '数据提取'
    },
    {
      agent_id: 'data-cleaner',
      task_template: '清洗异常值和缺失数据：\n{{previous_output}}',
      description: '数据清洗'
    },
    {
      agent_id: 'data-analyst',
      task_template: '计算统计指标（均值、方差、趋势）：\n{{previous_output}}',
      description: '统计分析'
    },
    {
      agent_id: 'chart-maker',
      task_template: '生成 K 线图和趋势图：\n{{previous_output}}',
      description: '可视化'
    }
  ]
})

// 执行流程（单次工具调用）：
// Step 1: data-extractor 提取 → "1000 行数据"
// Step 2: data-cleaner 清洗 → "980 行有效数据"
// Step 3: data-analyst 分析 → "均值 2.3%, 波动率 1.5%"
// Step 4: chart-maker 生成 → "已生成 chart.png"
```

### **示例 2: 代码审查（Sequential）**

```typescript
// LLM 决策使用 TeamTool - Sequential 策略
agent_team({
  strategy: 'sequential',
  goal: '全面审查 PR #456',
  members: [
    { agent_id: 'security-auditor', capabilities: ['安全漏洞扫描'] },
    { agent_id: 'performance-analyzer', capabilities: ['性能分析'] },
    { agent_id: 'code-style-checker', capabilities: ['代码规范检查'] }
  ]
})

// 执行流程：
// Step 1: security-auditor 执行
//   → "发现 2 个 XSS 漏洞，1 个 SQL 注入风险"
//
// Step 2: performance-analyzer 执行
//   输入包含: "[Step 1 - security-auditor] 发现 2 个 XSS 漏洞..."
//   → "发现 3 个 N+1 查询问题，建议优化数据库索引"
//
// Step 3: code-style-checker 执行
//   输入包含: 前两步的所有结果
//   → "基于前面的安全和性能问题，建议重构..."
```

### **示例 3: 智能推荐（MatchAgent）**

```typescript
// LLM 不确定用哪个 Agent
match_agent({
  task_description: '优化这个网站的加载速度',
  domain_hint: 'performance'
})

// 系统返回推荐：
// 1. performance-optimizer (95% match) - 专业性能优化
// 2. frontend-expert (72% match) - 前端优化经验
// 3. coder (45% match) - 通用编程能力

// LLM 基于推荐决策：
task({ agent_id: 'performance-optimizer', description: '...' })
```

## 🎉 总结

### **核心成果**

✅ **串行上下文传递**：完全自动化（Sequential 增强 + ChainTool）
✅ **决策辅助**：LLM 可查询和获取推荐（ListAgents + MatchAgent）
✅ **灵活性与智能化兼顾**：LLM 主导 + 系统辅助

### **架构优势**

- 🎯 **统一决策**：所有决策由 LLM 做出，系统不越权
- 🧠 **智能推荐**：多维度评分，提供最佳建议
- 🔗 **自动串联**：ChainTool 大幅简化工作流编排
- 📈 **可扩展**：未来可引入学习和完全自动化
