# 补充输入队列测试计划

## 测试目标

验证补充输入队列扩展功能的正确性、稳定性和用户体验。

## 核心功能测试

### 场景 1：单次补充输入

**目标**：验证单次补充与之前行为基本一致

**步骤**：
1. 启动 xuanji：`npm run dev`
2. 输入问题："介绍一下 TypeScript"
3. 等待 Agent 输出约 10 行后，输入："请简化"并按 Enter
4. 观察 UI 变化

**预期结果**：
- ✅ 第一轮 10 行内容归档到 Static
- ✅ 动态区域显示："✓ 已收到 1 条补充"
- ✅ 显示："最新：请简化"
- ✅ 第二轮内容流式输出
- ✅ 完成后，"请简化" 添加到历史
- ✅ UI 提示消失

### 场景 2：快速连续补充 3 次

**目标**：验证队列基本功能

**步骤**：
1. 启动 xuanji
2. 输入问题："解释递归"
3. 快速连续输入以下内容（每次按 Enter）：
   - "请举例"
   - "用 Python"
   - "请详细解释"
4. 观察 UI 变化

**预期结果**：

**第 1 次补充后**：
```
✓ 已收到 1 条补充
最新：请举例
```

**第 2 次补充后**：
```
✓ 已收到 2 条补充（按顺序处理）
最新：用 Python
其余 1 条将依次追加
```

**第 3 次补充后**：
```
✓ 已收到 3 条补充（按顺序处理）
最新：请详细解释
其余 2 条将依次追加
```

**完成后，Static 中按顺序显示**：
```
👤 请举例
💬 追加消息

👤 用 Python
💬 追加消息

👤 请详细解释
💬 追加消息
```

### 场景 3：队列满（超过 10 次补充）

**目标**：验证队列长度限制

**步骤**：
1. 启动 xuanji
2. 输入问题："写一篇长文"
3. 快速连续输入 11 次补充（编号 1-11）：
   - "补充1"
   - "补充2"
   - ...
   - "补充11"
4. 观察 UI 变化

**预期结果**：
- ✅ 前 10 次补充正常入队
- ✅ 第 11 次补充时，队列显示 "已收到 10 条补充"（第 1 条被移除）
- ✅ 完成后，Static 中只有 "补充2" 到 "补充11"（共 10 条）
- ✅ "补充1" 被移除（FIFO）

### 场景 4：连续相同内容去重

**目标**：验证去重功能

**步骤**：
1. 启动 xuanji
2. 输入问题："介绍 React"
3. 快速连续输入 3 次 "继续"（每次按 Enter）
4. 观察 UI 变化

**预期结果**：
- ✅ 第 1 次："继续" 入队，显示 "已收到 1 条补充"
- ✅ 第 2 次：与最后一条相同，不入队，仍显示 "已收到 1 条补充"
- ✅ 第 3 次：与最后一条相同，不入队，仍显示 "已收到 1 条补充"
- ✅ 完成后，Static 中只有 1 条 "继续"

### 场景 5：不同内容不去重

**目标**：验证去重只针对连续相同内容

**步骤**：
1. 启动 xuanji
2. 输入问题："介绍 React"
3. 快速连续输入：
   - "继续"
   - "简化"
   - "继续"（与第 1 次相同，但中间有其他内容）
4. 观察 UI 变化

**预期结果**：
- ✅ 所有 3 条都入队，显示 "已收到 3 条补充"
- ✅ 完成后，Static 中有 3 条："继续" → "简化" → "继续"

### 场景 6：Ctrl+C 中断时处理队列

**目标**：验证中断时的队列处理

**步骤**：
1. 启动 xuanji
2. 输入问题："写一篇长文"
3. 快速连续输入 3 次补充：
   - "停止"
   - "太长了"
   - "简化"
4. 立即按 Ctrl+C 中断
5. 观察 UI 变化

**预期结果**：
- ✅ 第一轮内容归档到 Static
- ✅ 所有 3 条补充按顺序添加到历史：
  ```
  👤 停止
  👤 太长了
  👤 简化
  ⏸️  会话已中断
  ```
- ✅ 队列清空（UI 提示消失）

### 场景 7：长响应 + 缓冲 + 多次补充

**目标**：验证缓冲模式下的队列功能

**步骤**：
1. 启动 xuanji
2. 输入问题："列举 100 个编程术语并解释"
3. 等待输出超过 50 行，进入缓冲模式
4. 在缓冲模式下快速连续输入 3 次补充
5. 观察 UI 变化

**预期结果**：
- ✅ 第一轮内容（100+ 行）归档到 Static
- ✅ 缓冲模式进度消失
- ✅ 队列提示显示 "已收到 3 条补充"
- ✅ 第二轮输出重新开始计算行数
- ✅ 完成后，所有补充按顺序添加到历史

### 场景 8：工具调用期间多次补充

**目标**：验证工具执行不受影响

**步骤**：
1. 启动 xuanji
2. 输入问题："读取 package.json 并分析依赖"
3. 在 Agent 调用 Read 工具时，快速连续输入：
   - "只关注 dependencies"
   - "忽略 devDependencies"
4. 观察 UI 变化

**预期结果**：
- ✅ 第一轮文本归档
- ✅ 工具调用结果正常显示
- ✅ 队列提示显示 "已收到 2 条补充"
- ✅ 第二轮输出基于所有补充内容调整

### 场景 9：空队列（无补充）

**目标**：验证边界情况

**步骤**：
1. 启动 xuanji
2. 输入问题："Hello"
3. 等待 Agent 响应完成（不补充输入）
4. 观察 UI 变化

**预期结果**：
- ✅ 无队列提示显示
- ✅ `pendingUserInputs.length === 0`
- ✅ onEnd 不处理 pending 队列
- ✅ 正常流程不受影响

### 场景 10：队列中的长内容

**目标**：验证长补充内容的展示

**步骤**：
1. 启动 xuanji
2. 输入问题："介绍 React"
3. 快速连续输入 2 次：
   - 第 1 次：一段很长的文本（超过 60 字符）
   - 第 2 次："继续"
4. 观察 UI 变化

**预期结果**：
- ✅ 队列提示显示 "已收到 2 条补充"
- ✅ 最新内容截断显示（60 字符 + "..."）
- ✅ 完成后，Static 中显示完整内容（不截断）

**预期**：✅ FIFO 正确工作

1. 输入问题

3. 验证不添加到队列

**预期**：✅ 空输入被忽略（handleSubmit 中 `input.trim()` 会过滤）
