# Xuanji GUI 设计方案 v1.0

> 设计日期：2026-03-11
> 目标：将 Xuanji CLI 升级为现代化的桌面 AI Agent 产品

---

## 一、设计理念

### 1.1 产品定位

**Shibit Xuanji Desktop** — 面向开发者的智能编程助手

- **核心价值**：AI 驱动的代码理解、生成、重构、调试
- **差异化**：本地记忆系统、会话时光倒流、子代理协作
- **目标用户**：专业开发者、技术 Lead、开源贡献者

### 1.2 设计原则

| 原则 | 说明 | 参考产品 |
|------|------|---------|
| **对话优先** | 以对话为中心，工具围绕对话展开 | Claude Desktop、ChatGPT |
| **上下文感知** | 自动感知项目、文件、Git 状态 | Cursor、Windsurf |
| **非侵入式** | 不替代现有编辑器，作为智能副驾驶 | GitHub Copilot Chat |
| **透明可控** | 所有 AI 操作可追溯、可回滚 | Claude Code |
| **高性能** | 流式渲染、虚拟滚动、增量更新 | VS Code |

### 1.3 核心 UX 目标

- ✅ **零学习成本**：熟悉聊天工具的用户可直接使用
- ✅ **快速响应**：流式输出首 token < 500ms，界面操作 < 100ms
- ✅ **清晰反馈**：所有 AI 操作显示进度、耗时、Token 用量
- ✅ **容错设计**：Checkpoint 时光倒流、文件快照、操作可撤销
- ✅ **专业美学**：深色主题、高对比度、专业配色

## 三、核心功能模块

### 3.1 对话区（中间主区域）

#### 3.1.1 消息类型

| 类型 | 样式 | 交互 |
|------|------|------|
| **用户消息** | 右对齐气泡、蓝色背景 | 右键菜单：编辑/删除/复制 |
| **助手消息** | 左对齐气泡、深灰背景 | 支持 Markdown 渲染、代码高亮、一键复制代码块 |
| **思考过程** | 折叠面板（Extended Thinking） | 点击展开/折叠，显示思考时长 |
| **工具调用** | 嵌入式卡片（图标+名称+参数） | 点击展开完整参数 JSON |
| **系统消息** | 居中灰色文本 | 如"已保存会话""已创建 Checkpoint" |

#### 3.1.2 流式输出效果

```
┌─────────────────────────────────────────┐
│ 助手正在回复...                         │
│                                         │
│ 这段代码主要做了以下几件事█              │  ← 打字机效果
│                                         │
│ 🧠 正在思考... (3.2s)  [脉冲动画]       │  ← 思考状态
│                                         │
│ 🛠️ 工具调用: read_file                 │
│    ├─ 参数: src/auth/login.ts          │
│    └─ [进度条 ▓▓▓▓▓░░░ 65%]            │  ← 执行中
└─────────────────────────────────────────┘
```

#### 3.1.3 补充输入交互

**场景**：AI 正在输出时，用户想补充说明

```
┌─────────────────────────────────────────┐
│ 助手正在回复...                         │
│ 我会使用 JWT 来实现认证...█             │
│                                         │
│ ┌─ 用户补充 ──────────────────┐         │
│ │ ✓ 已收到：用英文写注释        │  ← 悬浮提示
│ └──────────────────────────────┘         │
│                                         │
│ [立即中断，基于新指令重新生成...]       │
└─────────────────────────────────────────┘
```

**交互逻辑**：
1. 用户在输入框键入内容 → Enter 发送
2. 显示绿色悬浮提示"✓ 已收到补充：{内容}"
3. 立即中断当前输出（调用 `agentLoop.interrupt()`）
4. 已输出内容归档到 Static，新输出重新开始

## 四、关键交互流程

### 4.1 新建会话

```
用户点击"+ 新会话"
    ↓

    ↓
状态栏显示："已创建新会话"
```

```
用户点击"保存"按钮（Cmd+S）
    ↓

  ┌────────────────────────┐

    ↓

    ↓

```

```

    ↓
弹出对话框：
  ┌────────────────────────┐

## 七、技术栈建议

### 7.1 桌面框架选型

| 框架 | 优势 | 劣势 | 推荐度 |
|------|------|------|--------|
| **Electron** | 生态成熟、跨平台、VS Code 同框架 | 包体积大（~150MB） | ⭐⭐⭐⭐⭐ |
| **Tauri** | 包体积小（~10MB）、Rust 性能高 | 生态较新、Windows 兼容性问题 | ⭐⭐⭐⭐ |
| **Flutter Desktop** | 原生性能、统一代码库 | 学习曲线陡峭、生态不成熟 | ⭐⭐⭐ |

**推荐**：**Electron + React**（与 CLI 的 Ink/React 统一技术栈）

### 7.2 前端技术栈

```
Electron 25+
├─ React 18 (UI 框架)
├─ TypeScript (类型安全)
├─ Vite (构建工具)
├─ TailwindCSS (样式)
├─ Zustand (状态管理，轻量替代 Redux)
├─ React-Markdown (Markdown 渲染)
├─ Prism.js (代码高亮)
├─ ECharts (图表库)
├─ framer-motion (动画库)
└─ @tanstack/react-virtual (虚拟滚动)
```

### 7.3 架构设计

```
xuanji-desktop/
├─ main/                  # Electron 主进程
│  ├─ index.ts           # 主进程入口
│  ├─ window.ts          # 窗口管理
│  ├─ ipc.ts             # IPC 通信
│  └─ bridge.ts          # 与 CLI 核心模块桥接
│
├─ renderer/              # Electron 渲染进程（UI）
│  ├─ components/        # React 组件
│  │  ├─ ChatArea/       # 对话区组件
│  │  ├─ Sidebar/        # 侧边栏组件
│  │  ├─ RightPanel/     # 右侧面板组件
│  │  └─ ...
│  ├─ hooks/             # 自定义 Hooks
│  ├─ stores/            # Zustand 状态管理
│  └─ App.tsx            # 根组件
│
├─ shared/                # 共享模块
│  └─ types.ts           # 类型定义
│
└─ core/                  # 复用 CLI 核心模块（符号链接）
   ├─ agent/
   ├─ memory/
   ├─ tools/
   └─ ...
```

**关键点**：
- **复用 CLI 核心代码**：`core/` 模块通过符号链接或 npm workspace 共享
- **IPC 通信**：主进程运行 AgentLoop，渲染进程通过 IPC 发送消息和接收流式响应
- **状态管理**：Zustand 管理会话列表、Checkpoint、Token 用量等全局状态

### 7.4 性能优化策略

| 优化点 | 方案 | 预期效果 |
|--------|------|---------|
| **虚拟滚动** | @tanstack/react-virtual | 10,000 条消息无卡顿 |
| **增量渲染** | React.memo + useMemo | 减少 70% 不必要重渲染 |
| **代码高亮懒加载** | Prism.js 按需加载语言包 | 首屏加载快 50% |
| **流式输出** | Web Streams API | 首 token 延迟 < 500ms |
| **数据库索引** | SQLite FTS5 全文搜索 | 记忆检索 < 50ms |
