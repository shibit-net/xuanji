# 流式输出补充输入修复 - 测试文档

## 修复内容总结

### 修复1：interrupt 前归档流式文本
**文件**：`src/adapters/cli/App.tsx:1825-1873`

**修改**：
1. 将 debounce 延迟从 500ms 减少到 100ms（提升响应速度）
2. 在调用 `agentLoop.interrupt()` 之前：
   - 先调用 `archiveStreamText()` 归档已输出的流式文本
   - 处理 `pendingUserInputs` 队列，将所有补充输入添加到历史消息

**效果**：
- ✅ 流式输出的内容不再丢失（已归档到 Static 区域）
- ✅ 补充输入被正确处理（添加到历史消息）
- ✅ 上下文不再隔断

### 修复2：改进 UI 反馈
**文件**：`src/adapters/cli/App.tsx:2307-2310`

**修改**：
在 pending 队列提示中增加状态显示：
```
✓ 已收到 N 条补充（按顺序处理） ⏳ 中断流式输出中...
```

**效果**：
- ✅ 用户可以看到补充输入正在被处理
- ✅ 提高了用户体验和信心

## 问题排查

如果测试失败，检查：

1. **流式文本未归档**：
   - 检查 `archiveStreamText()` 是否被调用
   - 检查日志：`grep "archiveStreamText" ~/.xuanji/logs/core.log`

2. **补充输入丢失**：
   - 检查 `pendingUserInputs` 队列是否正确清空
   - 检查日志：`grep "pendingUserInputs" ~/.xuanji/logs/core.log`

3. **debounce 不生效**：
   - 检查 `interruptDebounceTimerRef` 是否正确设置
   - 增加日志输出，验证 debounce 逻辑

4. **上下文隔断**：
   - 检查消息序列，确认归档的内容在正确位置
   - 使用 `/sessions` 查看 session 文件

## 已知限制

1. **Debounce 延迟**：100ms 的延迟可能在极快输入时仍有感知
2. **队列合并窗口**：3 秒内的连续输入会被合并，可能不符合某些场景
3. **工具执行不中断**：工具执行期间的补充输入不会立即生效
