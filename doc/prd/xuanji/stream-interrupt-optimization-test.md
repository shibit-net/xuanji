# 流式输出优化测试计划

## 功能说明

优化了 Agent 流式输出期间用户补充输入的展示逻辑，避免阻断流式输出的视觉连续性。

### 核心改动

1. **延迟添加消息**：用户在执行期间输入时，不立即添加到消息历史，而是保存到 `pendingUserInput` 状态
2. **视觉提示**：在输入框上方显示绿色提示："✓ 已收到补充：{用户输入}"
3. **流式输出完成后添加**：在 `onEnd` 回调中将 pending 输入添加到消息历史
4. **中断时也处理**：在 `handleInterrupt`（Ctrl+C）时也会将 pending 输入添加到历史

### 代码变更

- `src/adapters/cli/App.tsx`
  - 新增 `pendingUserInput` 状态（L403-406）
  - 修改 `handleSubmit` 中的执行期间分支（L1745-1753）
  - 在 `onEnd` 回调中添加 pending 处理（L1092-1102）
  - 在 `handleInterrupt` 中添加 pending 处理（L750-761）
  - 添加 pending 输入提示渲染（L2198-2208）

## 测试场景

### 场景 1：流式输出期间补充输入

**步骤**：
1. 启动 xuanji：`npm run dev`
2. 发送一个需要较长时间响应的问题（例如："写一篇关于 AI 发展史的文章"）
3. 在 Agent 开始流式输出文本时，立即输入补充内容（例如："请重点介绍 Transformer 架构"）
4. 按 Enter 提交

**预期结果**：
- ✅ 流式输出不会被打断，继续正常显示
- ✅ 输入框上方出现绿色提示："✓ 已收到补充：请重点介绍 Transformer 架构"
- ✅ 流式输出完成后，补充内容被添加到消息历史
- ✅ 消息历史中显示：
  ```
  👤 请重点介绍 Transformer 架构
  💬 追加消息
  ```

### 场景 2：多次补充输入

**步骤**：
1. 启动 xuanji
2. 发送问题："解释什么是递归"
3. 在流式输出时补充："请举个例子"
4. 等待几秒后再补充："用 Python 代码展示"
5. 等待流式输出完成

**预期结果**：
- ✅ 只保留最后一次补充输入（"用 Python 代码展示"）
- ✅ 流式输出完成后，只有最后一次补充被添加到历史

> **注意**：当前实现只支持单个 pending 输入（后续可扩展为数组支持多次补充）

### 场景 3：Ctrl+C 中断

**步骤**：
1. 启动 xuanji
2. 发送问题："详细解释量子计算"
3. 在流式输出时补充："请简化说明"
4. 立即按 Ctrl+C 中断执行

**预期结果**：
- ✅ Agent 停止执行
- ✅ 补充内容被添加到消息历史
- ✅ 显示中断提示："⏸️ 会话已中断"
- ✅ 消息历史顺序：
  ```
  👤 请简化说明
  ⏸️  会话已中断
  ```

### 场景 4：工具调用期间补充输入

**步骤**：

### 场景 5：长文本补充

**步骤**：

### 场景 7：交互对话框激活时

**步骤**：

## 已知限制

1. **单个 pending 输入**：当前实现只保留最后一次补充输入，多次补充会覆盖
   - 未来可扩展为数组支持多次补充
   - 需要在 UI 中显示多个 pending 提示

2. **提示位置固定**：pending 提示在输入框上方，可能在某些场景下不够明显
   - 可考虑添加动画效果
   - 或在输入框内部显示（类似 placeholder）

3. **无撤销机制**：用户无法撤销 pending 输入
   - 可添加 Esc 键撤销功能

## 性能影响

- ✅ **减少渲染**：执行期间不调用 `setMessages`，避免整个组件树重新渲染
- ✅ **流式输出流畅**：流式文本区域完全不受用户输入影响
- ✅ **内存开销小**：只新增一个状态变量（`pendingUserInput`）

## 用户体验改进

- ✅ **视觉连续性**：流式输出不再被打断，阅读体验更好
- ✅ **明确反馈**：绿色提示告知用户输入已收到
- ✅ **符合预期**：与 Claude Code 行为一致
