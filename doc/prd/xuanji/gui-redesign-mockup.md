## 交互流程示例

### 流程 1: 查看和编辑 Agent 配置

```
1. 用户点击 Sidebar → "⚙️  配置" → "Agents"
   ↓
2. Workspace 切换到 AgentLibrary 视图
   ↓
3. 展示所有 Agents（内置 + 自定义）
   ↓
4. 用户点击某个 Agent 的 [查看详情]
   ↓
5. 打开 AgentDetailDialog（只读视图）
   ↓
6. 用户点击 [编辑]（仅自定义 Agent 可编辑）
   ↓
7. 打开 AgentEditor（可编辑表单）
   ↓
8. 用户修改配置 → 点击 [保存]
   ↓
9. 保存到配置文件 → 触发重载
   ↓
10. AgentLibrary 自动刷新
```

### 流程 2: 监控 Agent 运行状态

```
1. 用户在 ChatView 发送消息
   ↓
2. Agent 开始执行
   ↓
3. Inspector Panel 的 AgentMonitor 自动显示：
   - Agent 状态卡片（"正在思考..."）
   - 思考气泡（实时更新）
   ↓
4. Agent 调用工具（如 read_file）
   ↓
5. AgentMonitor 显示：
   - 当前工具调用（工具名、参数、状态）
   ↓
6. 工具调用完成
   ↓
7. ToolMonitor 更新：
   - 工具调用统计 +1
   - 最近调用列表顶部插入新记录
   ↓
8. Agent 返回结果
   ↓
9. AgentMonitor 显示："任务完成"
```

### 流程 3: 搜索和查看工具

```
1. 用户点击 Sidebar → "⚙️  配置" → "Tools"
   ↓
2. Workspace 切换到 ToolRegistry 视图
   ↓
3. 展示所有工具（按分类折叠）
   ↓
4. 用户在搜索框输入 "read"
   ↓
5. 过滤显示匹配的工具：
   - read_file
   ↓
6. 用户点击 [查看详情]
   ↓
7. 展开工具详情：
   - 描述
   - 输入参数 Schema
   - 属性（只读/可写、必备/可选）
   - 所属分类
   - 依赖该工具的 Skills
```
