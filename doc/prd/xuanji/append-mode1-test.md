# 补充输入方式1 - 测试文档

## 测试目标

验证"立即中断，清除重来"模式的正确性和用户体验。

---

## 测试环境

- xuanji 版本：0.9.0
- Node.js：20.x
- 终端：iTerm2 / Terminal.app
- LLM：[CC]claude-opus-4-6

---

## 核心测试场景

### TC1：基础补充 - 切换语言

**步骤**：
1. 启动 xuanji：`npm run dev`
2. 输入：`总结一下 xuanji 项目的核心功能`
3. 等待流式输出开始（观察中文输出）
4. 当输出约 50-100 字时，输入补充：`使用英文总结`

**预期结果**：
- ✅ 中文输出立即停止（100ms 后）
- ✅ 中文内容从 UI 中**清除**（不保留）
- ✅ 显示补充输入消息：`使用英文总结`（普通 user 消息，无 💬 标记）
- ✅ 显示处理中提示：`⏳ 正在处理补充输入... 使用英文总结`
- ✅ 新的流式输出开始（英文）
- ✅ 处理中提示消失
- ✅ 英文输出是**完整的**项目总结（不是中文的翻译，而是重新生成）
- ✅ 历史消息中只有英文输出，无中文残留

**验证方法**：
```bash
# 1. 观察 UI：中文内容应被清除
# 2. 检查历史消息（/sessions 或 session 文件）
grep "xuanji 项目" ~/.xuanji/sessions/*.jsonl  # 应无结果（中文被清除）
```

---

### TC2：快速连续补充

**步骤**：
1. 输入：`介绍一下 TypeScript 的特性`
2. 等待流式输出开始
3. 快速连续输入（间隔 < 50ms）：
   - `用中文简单介绍`
   - `不，用日语详细介绍`

**预期结果**：
- ✅ 显示队列提示：`✓ 已收到 2 条补充（按顺序处理）`
- ✅ 两条输入在 3 秒内合并为一条：`用中文简单介绍\n\n不，用日语详细介绍`
- ✅ 只触发一次 interrupt（检查日志）
- ✅ 基于最后的指令生成**日语详细介绍**
- ✅ 无中文输出残留

**验证方法**：
```bash
# 检查 API 调用次数
grep "interrupt" ~/.xuanji/logs/core.log | tail -5
# 应只看到一次 interrupt 调用
```

---

### TC3：补充输入后立即 Ctrl+C

**步骤**：
1. 输入：`生成一个测试报告`
2. 等待流式输出开始
3. 输入补充：`使用表格格式`
4. 立即按 Ctrl+C（在 debounce 100ms 内）

**预期结果**：
- ✅ 显示中断提示：`⏸️ 会话已中断`
- ✅ 补充输入被添加到历史：`使用表格格式`
- ✅ 无补充输入丢失
- ✅ processingAppend 状态被清除

**验证方法**：
```bash
# 检查 session 文件
tail -20 ~/.xuanji/sessions/*.jsonl
# 应看到 "使用表格格式" 消息
```

---

### TC4：工具执行期间补充

**步骤**：
1. 输入：`读取 package.json 文件并总结依赖`
2. 等待工具开始执行（看到 "🔧 read_file" 提示）
3. 在工具执行期间输入：`只显示 dependencies，不要 devDependencies`

**预期结果**：
- ✅ 工具继续执行（不中断）
- ✅ 显示队列提示：`✓ 已收到 1 条补充`（无"中断"字样）
- ✅ 工具执行完成后，补充输入被处理
- ✅ 补充输入显示为普通 user 消息
- ✅ Agent 基于工具结果 + 补充生成回复

**验证方法**：
- 观察工具执行是否完整
- 检查回复内容是否符合补充要求

---

### TC5：连续多次补充

**步骤**：
1. 输入：`介绍一下 React`
2. 等待流式输出开始
3. 输入补充1：`用中文介绍`
4. 等待新输出开始（约 1-2 秒）
5. 输入补充2：`添加示例代码`
6. 等待新输出开始
7. 输入补充3：`用日语总结`

**预期结果**：
- ✅ 第一次补充：
  - 英文输出被清除
  - 显示 `用中文介绍`
  - 显示处理中提示
  - 重新生成中文介绍
- ✅ 第二次补充：
  - 中文介绍被清除
  - 显示 `添加示例代码`
  - 显示处理中提示
  - 重新生成带示例的介绍
- ✅ 第三次补充：
  - 带示例的介绍被清除
  - 显示 `用日语总结`
  - 显示处理中提示
  - 重新生成日语总结
- ✅ 历史消息清晰：每次补充都是独立的 user 消息
- ✅ 最终只保留日语总结输出

**验证方法**：
```bash

```

## 已知问题

### 问题1：已输出内容丢失

**现象**：
- 补充输入后，已输出的大量内容被清除
- 用户可能觉得"浪费"

**状态**：
- 这是方式1 的设计特性，非 bug
- 适合"改变方向"的补充，不适合"追加内容"

**缓解**：
- 文档中说明适用场景
- 建议用户等输出完成后再追加内容

---

### 问题2：100ms debounce 内容残留

**现象**：
- 补充输入后，内容仍显示 100ms

**状态**：
- 设计决策，避免频繁 API 调用
- 100ms 几乎无感知

**缓解**：
- 显示"✓ 已收到补充"提示，告知已接收
