# 统一工作台设计方案

## 愿景

将右侧面板从"多个独立 Tab"改造成"统一的工作台视图"，让用户像看真实的工作台一样，一目了然地看到 Xuanji 的整个工作轨迹。

## 设计理念

### 从分散到统一

**当前问题**：
- 6 个独立的 Tab（工作区、Agent、工具、上下文、记忆、日志）
- 需要频繁切换才能了解全貌
- 信息分散，缺乏整体感
- 难以追踪完整的工作流程

**新设计目标**：
- 统一的时间轴视图
- 所有事件按时间顺序展示
- 一目了然的工作轨迹
- 像看真实工作台一样直观

### 核心概念

**工作台 = 时间轴 + 事件卡片 + 状态面板**

## 事件卡片类型

### 1. 思考卡片（Thinking Card）

```tsx
┌─────────────────────────────────────┐
│ 💭 Agent 正在思考                    │
│ ─────────────────────────────────── │
│ "我需要先读取配置文件来了解项目结构，│
│  然后分析依赖关系..."                │
│                                     │
│ Agent: Xuanji Assistant             │
│ 状态: 🟡 思考中                     │
└─────────────────────────────────────┘
```

**数据来源**：`runtimeStore.agentStatus.currentThought`

### 2. 工具卡片（Tool Card）

```tsx
┌─────────────────────────────────────┐
│ 🔧 Read                              │
│ ─────────────────────────────────── │
│ 📄 src/config.json                   │
│                                     │
│ ✓ 完成 · 120ms                      │
│ [点击查看详情] →                    │
└─────────────────────────────────────┘

执行中状态：
┌─────────────────────────────────────┐
│ 🔧 Read                              │
│ ─────────────────────────────────── │
│ 📄 src/config.json                   │
│                                     │
│ 🔄 执行中... · 已用时 1.2s          │
└─────────────────────────────────────┘

失败状态：
┌─────────────────────────────────────┐
│ 🔧 Read                              │
│ ─────────────────────────────────── │
│ 📄 /path/not/found.txt               │
│                                     │
│ ❌ 失败 · 文件不存在                │
│ [查看错误详情] →                    │
└─────────────────────────────────────┘
```

**数据来源**：`executionStore.toolExecutions`

**交互**：
- 点击卡片展开详情（input/output）
- 折叠/展开切换

### 3. 回复卡片（Response Card）

```tsx
┌─────────────────────────────────────┐
│ ✨ 回复生成                          │
│ ─────────────────────────────────── │
│ "根据配置文件分析，项目使用了        │
│  TypeScript + React + Vite 技术栈..." │
│                                     │
│ Token: 245 (↑180 ↓65)               │
│ Cost: $0.002                        │
└─────────────────────────────────────┘
```

**数据来源**：
- `runtimeStore.messageStream.text`
- `runtimeStore.tokenUsage`
- `runtimeStore.cost`

### 4. 上下文卡片（Context Card）

```tsx
┌─────────────────────────────────────┐
│ 📂 上下文更新                        │
│ ─────────────────────────────────── │
│ 关注文件:                           │
│ + src/config.json                   │
│ + src/App.tsx                       │
│ + package.json                      │
│                                     │
│ 工作目录: ~/project                 │
└─────────────────────────────────────┘
```

**数据来源**：`runtimeStore.contextInfo`

### 5. 日志卡片（Log Card）

```tsx
┌─────────────────────────────────────┐
│ 📝 系统日志                          │
│ ─────────────────────────────────── │
│ [INFO] 开始处理用户消息              │
│ [DEBUG] 加载配置文件成功             │

│                                     │
```

**数据来源**：`runtimeStore.logs`

**优化**：
- 默认折叠，只显示最近 3 条
- 点击展开查看全部

### 6. 记忆卡片（Memory Card）

```tsx
┌─────────────────────────────────────┐
│ 🧠 检索到相关记忆                    │

│ [查看详情] →                        │

```

**数据来源**：`historyStore.memoryEntries`

**状态指示器**：

## 时间轴设计

### 视觉样式

```
垂直时间轴：

    │
    ●────── 20:15:32 ────────
    │       思考卡片
    │
    ●────── 20:15:35 ────────
    │       工具卡片
    │
    ●────── 20:15:38 ────────
    │       回复卡片
    │
    ▼
```

### 时间戳格式

- 同一分钟内：只显示秒（如 `:32`）
- 不同分钟：显示分:秒（如 `15:32`）
- 不同小时：显示时:分:秒（如 `20:15:32`）

### 自动滚动

- 新事件出现时自动滚动到底部
- 用户手动滚动时暂停自动滚动
- 显示"跳转到最新"按钮

## 布局设计

### 三段式布局

```
┌─────────────────────────────────────┐
│ 📊 状态面板（固定顶部）              │ ← 80px 固定高度
├─────────────────────────────────────┤
│                                     │
│                                     │
│ 📜 时间轴 + 事件卡片                │ ← flex-1 可滚动
│    （主要内容区域）                 │
│                                     │
│                                     │
├─────────────────────────────────────┤
│ 🔍 过滤器（可选，可折叠）            │ ← 40px 可折叠
└─────────────────────────────────────┘
```

### 响应式设计

- 宽度 < 300px：卡片简化显示
- 宽度 300-400px：正常显示
- 宽度 > 400px：展开详情

## 数据模型

### 统一事件模型

```typescript
interface WorkspaceEvent {
  id: string;
  timestamp: number;
  type: 'thinking' | 'tool' | 'response' | 'context' | 'log' | 'memory';
  data: ThinkingEvent | ToolEvent | ResponseEvent | ContextEvent | LogEvent | MemoryEvent;
}

interface ThinkingEvent {
  thought: string;
  agentName: string;
  status: 'thinking' | 'done';
}

interface ToolEvent {
  toolName: string;
  input: Record<string, unknown>;
  output?: string;
  status: 'running' | 'success' | 'error';
  duration?: number;
  error?: string;
}

interface ResponseEvent {
  content: string;
  tokenUsage: { input: number; output: number; cached: number };
  cost: number;
}

interface ContextEvent {
  focusedFiles: string[];
  recentFiles: string[];
  workingDirectory: string;
}

interface LogEvent {
  level: 'debug' | 'info' | 'warn' | 'error';
  category: string;
  message: string;
  data?: any;
}

interface MemoryEvent {
  content: string;
  type: string;
  relevance: number;
}
```

## 兼容性考虑

### 渐进式迁移

**方案 A：完全替换**
- 移除所有 Tab，只保留统一工作台
- 优点：简洁统一
- 缺点：丢失详细查看能力

**方案 B：双模式**
- 默认显示统一工作台
- 保留 Tab 作为详细查看入口
- 优点：兼顾整体和细节
- 缺点：稍微复杂

**推荐方案 B**：
```
┌─────────────────────────────────────┐
│ [工作台] [详细] ← 模式切换           │
├─────────────────────────────────────┤
│                                     │
│ 工作台模式：统一时间轴视图           │
│                                     │
│ 或                                  │
│                                     │
│ 详细模式：传统 Tab 视图              │
│                                     │
└─────────────────────────────────────┘
```
