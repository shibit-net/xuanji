# 璇玑（Xuanji）商业计划书

---

## 1. 执行摘要

### 1.1 项目定位
**璇玑（Xuanji）** — 面向全场景的 Agentic OS，不是一个聊天机器人，而是住在用户电脑里、越用越懂你的 AI 管家。

**双轮驱动产品矩阵**：
- **璇玑客户端**：面向个人和企业的多智能体协作平台（免费）
- **天工坊**：国内首个 MCP 和 Skills 开源社区与交易市场

### 1.2 核心价值
- **个人用户**：高粘性 AI 管家，知识图谱记忆、多 Agent 协作、动态 Prompt 优化
- **企业用户**：可定制的智能体协作系统，私有化部署
- **开发者**：天工坊平台发布 MCP/Skill，获得持续收入
- **算力提供者**：P2P 智能体网络，贡献算力获得收益

### 1.3 商业模式
**三层盈利结构**：
1. **产品免费**：璇玑客户端永久免费，扩大用户基数
2. **天工坊交易分成**：MCP/Skill 订阅/调用付费，平台抽取佣金
3. **P2P 智能体网络**：任务分发与算力交易，网络手续费

### 1.4 融资需求
- **融资金额**：500-1000 万元人民币（天使轮）
- **资金用途**：
  - 团队扩张：招聘后端、前端、产品、运营（5-8 人）
  - 市场推广：品牌建设、用户获取、社区运营
  - 产品迭代：加快功能开发、天工坊建设
  - 基础设施：服务器、云服务、安全合规

## 2. 市场分析

### 2.1 行业背景
全球 AI 助手市场正在快速爆发：
- **多智能体协作**：从单 Agent 到多 Agent 协同是必然趋势
- **Agentic OS**：AI 操作系统将成为下一代计算平台
- **MCP/Skill 生态**：类似 App Store 的智能体应用市场正在形成
- **数据隐私**：本地优先、隐私保护的 AI 产品需求激增

### 2.2 目标市场

#### 2.2.1 B2C 个人用户市场
- **规模**：中国数亿 AI 工具潜在用户
- **需求**：高效助手、个性化服务、本地隐私
- **痛点**：现有 AI 记不住用户、不会协作、Prompt 太臃肿

#### 2.2.2 B2B 企业用户市场
- **规模**：千万家中小企业需要 AI 升级
- **需求**：自动化工作流、降本增效、私有化部署
- **痛点**：定制化成本高、数据安全顾虑、集成复杂

#### 2.2.3 开发者市场
- **规模**：数百万 AI 开发者需要分发渠道
- **需求**：变现平台、用户触达、社区协作
- **痛点**：国际平台 ClawHub 在国内不可用，缺乏国内社区

### 2.3 竞争格局

| 竞争对手 | 优势 | 劣势 | 璇玑机会 |
|---------|------|------|---------|
| **OpenClaw** | 技术领先、ClawHub 生态 | 国外产品、国内不可用 | 本土化替代 |
| **Hermes Agent** | 开源社区活跃 | 终端界面、记忆系统弱 | 可视化+知识图谱 |
| **GitHub Copilot** | 大厂背书、用户基数大 | 专注代码、单 Agent | 全场景+多 Agent |
| **钉钉/飞书 AI** | 企业集成深 | 封闭生态、通用性弱 | 开放平台+P2P网络 |

### 2.4 市场机会
- **国产替代**：国际形势下，国内需要自主可控的 Agent 平台
- **生态空缺**：中国没有类似 ClawHub 的 MCP/Skill 社区
- **P2P 创新**：智能体网络是全新商业模式，具有先发优势

## 3. 产品介绍

### 3.1 璇玑核心功能

#### 3.1.1 🧠 知识图谱记忆系统
- 实体-关系-事件模型，信息真正"连起来"
- 用户反馈驱动，一次纠正，永久生效
- Cytoscape 可视化，直观看到知识网络

#### 3.1.2 🚀 多智能体协作引擎
- 5 种策略：串行/并行/层级/辩论/流水线
- 10+ 内置 Agent，无合适 Agent 时当场创建
- React Flow 实时可视化协作流程

#### 3.1.3 📚 L0-L2 动态 Prompt
- L0 基础层（始终加载）：身份、安全
- L1 场景层（按需加载）：10+ 专业场景
- L2 协调层（复杂任务加载）：多 Agent 协作规则
- **核心优势**：用户长期使用后，场景和 Skill 积累越多，LLM 越专注，Token 开销越低

#### 3.1.4 🔒 隐私与安全
- 所有数据本地存储，零遥测
- API Key 加密保存
- 双层安全架构：LLM 审计 + 硬编码守卫

### 3.2 天工坊平台
- **MCP 市场**：类似 npm，开发者发布和安装 MCP
- **Skill 商店**：类似 App Store，用户订阅或按次购买 Skill
- **交易分成**：平台抽取佣金（建议 20-30%）
- **开源社区**：GitHub 风格的协作与讨论

### 3.3 P2P 智能体网络
- **任务分发**：用户可将任务分发到网络
- **算力交易**：算力提供者自主定价
- **支付系统**：积分或代币结算
- **网络效应**：用户越多，网络越强大

### 3.4 技术栈
| 层级 | 技术 |
|------|------|
| 语言 | TypeScript 5.7+ |
| 运行时 | Node.js 20+ |
| 桌面 | Electron + React + TailwindCSS |
| 可视化 | React Flow + Cytoscape |
| 数据库 | SQLite |

### 3.5 产品路线图

| 阶段 | 时间 | 目标 |
|------|------|------|
| **公测期** | Now-2026 Q3 | 用户增长至 1000+，打磨产品体验 |

| **规模扩张** | 2027 Q3+ | 用户破 10 万，生态繁荣 |

## 4. 商业模式

### 4.1 三层盈利结构

#### 第一层：免费获客（基础层）
- 璇玑客户端完全免费
- 目标：快速扩大用户基数，形成网络效应
- 预期：2027 年用户 10 万+

#### 第二层：天工坊交易分成（生态层）
**收入来源**：
1. **MCP/Skill 订阅费**：开发者定价，平台抽成 20-30%
2. **按次调用费**：按使用次数付费，平台抽成
3. **天工坊 Pro**：高级功能订阅（如数据分析、推广位）

**定价策略**：
- 免费 MCP/Skill：鼓励生态繁荣
- 付费 MCP/Skill：开发者自主定价，0.99-99 元/月
- 调用费：0.01-0.1 元/次

#### 第三层：P2P 智能体网络（网络层）
**收入来源**：
1. **网络交易手续费**：每笔交易抽成 5-10%
2. **高级节点服务**：为企业提供可靠节点服务
3. **网络治理代币**（未来）

**定价策略**：
- 节点提供者自主定价
- 平台提供价格指导和信誉系统

### 4.2 客户获取策略
- **B2C 用户**：技术社区推广（V2EX、掘金、GitHub）、KOL 测评、口碑传播
- **B2B 客户**：企业试用、销售对接、定制化方案
- **开发者**：竞赛激励、早期入驻扶持、收入分成倾斜

### 4.3 增长飞轮
```
更多用户 → 更多需求 → 更多开发者 → 更多 MCP/Skill → 产品更强大 → 更多用户
```

## 8. 风险分析与应对

### 8.1 技术风险
- **风险**：LLM 能力瓶颈、多 Agent 协作不稳定
- **应对**：持续迭代优化、灰度发布、备选方案

### 8.2 市场风险
- **风险**：用户接受度、竞争加剧
- **应对**：快速验证产品、构建生态壁垒、差异化定位

### 8.3 运营风险
- **风险**：社区冷启动、开发者增长慢
- **应对**：早期激励计划、合作伙伴引入、官方示范内容

### 8.4 政策风险
- **风险**：AI 监管、数据合规
- **应对**：本地优先架构、合规审核、法务咨询
