# 岳飞《满江红·怒发冲冠》深度分析报告

> 怒发冲冠，凭栏处、潇潇雨歇。抬望眼，仰天长啸，壮怀激烈。
> 三十功名尘与土，八千里路云和月。莫等闲，白了少年头，空悲切！
> 靖康耻，犹未雪。臣子恨，何时灭！驾长车，踏破贺兰山缺。
> 壮志饥餐胡虏肉，笑谈渴饮匈奴血。待从头、收拾旧山河，朝天阙。

## 一、创作背景

### 1.1 历史坐标
- **靖康之变（1127年）**：金兵破汴京，徽钦二帝被掳，北宋灭亡
- **岳飞生平**：1103-1142，南宋抗金名将，39岁以"莫须有"罪名遇害
- **创作时间**：约1132-1136年间（学术界有争议），正值岳飞北伐受挫或备战之际

### 1.2 军事背景
此词非文人书斋之作，而是**前线统帅的作战宣言**。绍兴四年（1134），岳飞收复襄汉六郡；绍兴六年（1136），岳飞第二次北伐，孤军深入，粮尽援绝，被迫撤回鄂州。词中"三十功名尘与土"恰指此役。

## 二、逐句解读

**上片：英雄自白**
- "怒发冲冠"——典出《史记·蔺相如列传》"相如因持璧却立倚柱，怒发上冲冠"
- "潇潇雨歇"——既是实景，也是象征：暴风雨已过，但国家风雨未歇
- "三十功名尘与土"——视功名如尘土，大气磅礴，超越个人得失
- "八千里路云和月"——概括转战南北之艰辛，云月为伴，悲壮苍凉
- "莫等闲，白了少年头"——千古励志名言，催人奋进

**下片：复仇誓言**
- "靖康耻，犹未雪"——从个人到国家，主题升华
- "驾长车，踏破贺兰山缺"——贺兰山为金人腹地，目标直指大本营
- "壮志饥餐胡虏肉"——极言仇恨之深，非残暴而是血性
- "待从头、收拾旧山河"——收复失地之决心
- "朝天阙"——对朝廷之忠，反衬后来被冤杀之悲

## 三、核心思想

1. **家国情怀**：从个人功名到民族大义
2. **抗争精神**：面对强敌不屈不挠
3. **时间紧迫感**："莫等闲"——时不我待
4. **忠君爱国**：虽悲愤仍"朝天阙"

## 四、文学特色

1. **豪放风格**：开豪放词之先河，情感外放、气势磅礴
2. **虚实结合**：实写当前处境，虚写未来理想
3. **对比手法**：尘与土、云和月、饥餐与笑谈
4. **用典自然**：怒发冲冠（蔺相如）、贺兰山

## 五、历史影响与当代价值

- **古代**：激励南宋军民抗金斗志
- **近代**：抗战时期成为中华儿女的精神旗帜
- **当代**：入选中学语文教材，成为爱国主义教育经典
- **世界**：被译为多国文字，是人类反抗压迫的精神象征

# 南宋三大家诗词对比研究

## 岳飞《满江红》· 李清照《声声慢》· 辛弃疾《定风波》

### 历史坐标
- 靖康之变 (1127年) → 南宋建立
- 岳飞 (1103-1142)：战死/冤杀
- 李清照 (1084-1155)：流亡
- 辛弃疾 (1140-1207)：闲居

### 三种回应
- 岳飞：怒发冲冠 → 向外 → 战
- 李清照：寻寻觅觅 → 向内 → 泣
- 辛弃疾：休恨 → 超越 → 思

### 文学风格
- 岳飞：豪放悲壮
- 李清照：婉约凄切
- 辛弃疾：沉郁顿挫

### 当代启示
- 面对困境的三种姿态：抗争、承受、超越
- 三者互补，共同构成完整的精神图谱
# 通知循环分析报告

## 现象
子agent在完成任务后，持续收到"后台任务已完成，请汇总结果告知用户"通知，陷入无限循环。

## 根因
`src/core/agent/AgentLoop.ts` - `autoSummarizeTaskCompletion()` 方法：

```typescript
private async autoSummarizeTaskCompletion(): Promise<void> {
    this.callbacks?.onAutoSummarize?.();
    this._isAutoSummarizeRun = true;
    try {
        await this.run('[系统通知] 后台任务已完成，请汇总结果告知用户');
    } finally {
        this._isAutoSummarizeRun = false;
    }
}
```

## 循环链路
1. 子agent完成任务并回应
2. 系统标记任务为 `completed`
3. `autoSummarizeTaskCompletion()` 被调用
4. `this.run('...')` 重新执行主Agent Loop
5. 主Agent Loop将通知发送给LLM（子agent）
6. LLM回应 → 回到步骤1

## 关键变量
- `_isAutoSummarizeRun`：在 `finally` 中重置为 `false`，允许循环重复
- `_pendingTaskCompletions`：累积完成事件

## 建议修复
- 在 `autoSummarizeTaskCompletion()` 入口处检查 `_isAutoSummarizeRun`，如已为 `true` 则跳过
- 或限制子agent对此通知的响应次数
