# 通知循环分析报告

## 现象
子agent在完成任务后，持续收到"后台任务已完成，请汇总结果告知用户"通知，陷入无限循环。

## 根因
`src/core/agent/AgentLoop.ts` - `autoSummarizeTaskCompletion()` 方法：

```typescript
private async autoSummarizeTaskCompletion(): Promise<void> {
    this.callbacks?.onAutoSummarize?.();
    this._isAutoSummarizeRun = true;
    try {
        await this.run('[系统通知] 后台任务已完成，请汇总结果告知用户');
    } finally {
        this._isAutoSummarizeRun = false;
    }
}
```

## 循环链路
1. 子agent完成任务并回应
2. 系统标记任务为 `completed`
3. `autoSummarizeTaskCompletion()` 被调用
4. `this.run('...')` 重新执行主Agent Loop
5. 主Agent Loop将通知发送给LLM（子agent）
6. LLM回应 → 回到步骤1

## 关键变量
- `_isAutoSummarizeRun`：在 `finally` 中重置为 `false`，允许循环重复
- `_pendingTaskCompletions`：累积完成事件

## 建议修复
- 在 `autoSummarizeTaskCompletion()` 入口处检查 `_isAutoSummarizeRun`，如已为 `true` 则跳过
- 或限制子agent对此通知的响应次数
