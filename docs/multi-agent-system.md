# 多智能体协作系统

璇玑的核心能力之一是组织多个智能体协同工作。以下是详细的系统介绍。

---

## 协作策略

璇玑支持 5 种协作策略，适用于不同场景：

### 1. 串行协作 (Sequential)

**适用场景**：任务需要按顺序完成，前一个任务的输出是后一个任务的输入。

**工作方式**：
- Agent 1 完成任务 A
- Agent 2 使用任务 A 的输出完成任务 B
- Agent 3 使用任务 B 的输出完成任务 C
- 依此类推...

**示例**：
```
需求分析 → 架构设计 → 代码实现 → 测试 → 文档
```

---

### 2. 并行协作 (Parallel)

**适用场景**：多个独立的任务可以同时进行，互不干扰。

**工作方式**：
- 多个 Agent 同时开始工作
- 每个 Agent 处理一个独立的子任务
- 所有任务完成后汇总结果

**示例**：
```
      ┌→ 前端开发
需求 ─┼→ 后端开发
      └→ 数据库设计
```

---

### 3. 层级协作 (Hierarchical)

**适用场景**：复杂任务需要一个协调者（Leader）来分配任务、汇总结果。

**工作方式**：
- Leader Agent 分析任务，分解成子任务
- Leader 分配子任务给 Worker Agent
- Worker Agent 执行任务并返回结果
- Leader 汇总结果，可能再分配新任务

**示例**：
```
      Leader
      /  |  \
   开发  设计  测试
```

---

### 4. 辩论协作 (Debate)

**适用场景**：需要多方观点碰撞，通过辩论找到最佳方案。

**工作方式**：
- Agent A 提出方案 A
- Agent B 提出方案 B
- Agent C 提出方案 C
- 各 Agent 互相辩论，指出优缺点
- 最终汇总达成共识或选择最佳方案

**示例**：
```
Agent1: 选 React!
Agent2: 选 Vue!
Agent3: 选 Svelte!
→ 辩论后选择最佳方案
```

---

### 5. 流水线协作 (Pipeline)

**适用场景**：任务可以分解成多个阶段，每个阶段专注一件事。

**工作方式**：
- 阶段 1：信息收集
- 阶段 2：分析处理
- 阶段 3：方案制定
- 阶段 4：执行实施

**示例**：
```
收集 → 分析 → 方案 → 执行
```

## 核心组件

### TeamManager（团队管理器）

负责：
- 创建和管理 Agent 团队
- 选择合适的协作策略
- 协调 Agent 之间的通信
- 监控任务执行状态

### WorktreeManager（工作区管理器）

负责：
- 为每个 Agent 创建独立的 Git Worktree
- 隔离不同 Agent 的工作环境
- 避免代码冲突
- 合并各 Agent 的工作成果

### SubAgentContext（子 Agent 上下文）

负责：
- 管理子 Agent 的状态
- 传递任务信息
- 收集执行结果
- 处理超时和错误

## 可视化界面

桌面应用提供了直观的协作可视化：

### React Flow 协作图

- 实时展示 Agent 协作流程
- 节点表示 Agent，边表示任务流转
- 动态更新任务状态
- 支持缩放和拖拽

### 任务监控面板

- 查看每个 Agent 的当前任务
- 显示任务进度和状态
- 实时日志输出
- 任务完成后自动切换

## 最佳实践

### 如何选择协作策略

| 任务类型 | 推荐策略 |
|---------|---------|
| 简单的顺序任务 | 串行协作 |
| 多个独立任务 | 并行协作 |
| 复杂的项目管理 | 层级协作 |
| 技术方案选择 | 辩论协作 |
| 多阶段复杂流程 | 流水线协作 |

### Agent 角色设计建议

- **Leader**：视野开阔，擅长统筹规划
- **Engineer**：技术深入，擅长代码实现
- **Designer**：注重体验，擅长界面设计
- **QA**：细心谨慎，擅长发现问题
- **Researcher**：好学不倦，擅长资料整理
