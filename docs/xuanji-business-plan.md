## 一、我们做什么

### 产品定位

璇玑由三个项目构成，全部由创始人独立开发（已验证：源码审查）：

| 项目 | 技术栈 | 规模 | 做什么 |
|------|--------|------|--------|
| **xuanji** | Node.js / Electron | 84,890 行 TS · 448 commits · 425 源文件 | 桌面 Agent 客户端 |
| **Starship** | Java / Spring Boot 3.4.3 | 318 文件 · 90 commits | 后端服务 + 天工坊完整 API |
| **shibit-web** | Vue 3 / Element Plus | 159 文件 · 60 commits | 官网 + 天工坊前端 |
| **合计** | — | **598 commits · 902 源文件** | |

### 三大核心壁垒

#### 1. 动态 Prompt 架构 — Token 消耗降低 75-92%

一个重度用户接入 50 个 MCP、安装 20 个 Skill、配置自定义 Prompt 后，全量常驻的系统 Prompt 膨胀到 **100K+ Token/次**。日均 50 次对话 = **500 万 Token/天**，不仅费用爆炸，Agent 还会「注意力涣散」。

璇玑的三层动态架构（L0/L1/L2）按需加载：

| 层级 | 加载策略 | Token | 场景 |
|------|---------|-------|------|
| **L0** | 始终加载 | ~5K | 核心身份、安全基线、协调协议 |
| **L1** | 按需注入 1-3 个 Scene | ~3-8K | 写代码 / 查 bug / 做计划 |
| **L2** | 仅复杂任务触发 | ~5-15K | 多 Agent 协作、架构分析 |

**实测对比**：

| 场景 | 全量常驻 | 璇玑动态 | 降幅 |
|------|---------|---------|------|
| 普通对话 | 100K+ Token | ~8K Token | **92%** |
| 复杂任务 | 100K+ Token | ~25K Token | **75%** |

按日均 50 次对话（30 次普通 + 20 次复杂）计算：全量常驻月均 ~¥800（以 DeepSeek 价格计），璇玑动态架构月均 ~¥65，**节省 92%**。对比 OpenAI/Claude 价位，节省幅度更大。

#### 2. 知识图谱记忆 — 切换成本随使用时长指数增长

璇玑的记忆不是对话历史，是**结构化知识图谱**——实体、关系、事件随使用时间自动累积。

**Before & After**：

> **打开一个新 AI 助手**：*"我是做什么的？上次项目改到哪了？那个 bug 的复现步骤是什么？"* → 每次对话从零开始，需要手动同步上下文。

> **打开璇玑（使用 3 个月后）**：*"你上次在 xuanji 项目修 AgentStateMachine 的 ghost node bug，改了 EventAdapter.ts 和 AgentStateMachine.ts 两个文件，root cause 是 agentId 和 teamId 不匹配。"* → 直接进入工作状态。系统已自动记住：几十个项目、几百个实体、数千条关联关系。

切换到另一个 AI 助手，不是换一个工具——是丢掉三个月亲手构建的整个数字认知世界。切换成本随使用时长指数增长。

#### 3. 模型无关架构 — 不参与军备竞赛的结构性优势

璇玑不提供大模型，用户自行购买 API Key 接入。这带来了三个结构性优势：

- **大模型越卷，璇玑越强**。模型能力提升和价格下降直接转化为璇玑用户体验提升和成本下降——璇玑不需要花一分钱参与模型军备竞赛
- **用户不被厂商锁定**。大厂绑定自己的全产业链（字节→豆包+DeepSeek，阿里→千问+DeepSeek），璇玑让用户自由选择能力最强的国际模型（Claude/GPT）或性价比最高的国内模型（DeepSeek/千问）
- **天然杠杆**。模型降价（行业趋势明确）→ 用户体验提升 + 用户支出降低 → 平台吸引力增强

### 天工坊：已有的生态基建

天工坊不是规划中的市场，后端和前端已大量开发并上线运行（已验证：源码审查）。当前 33 个 MCP/Skill 包可一键安装，均为创始人从开源社区收集的认可度较高的产品集成上线。

**已建成**：包管理（16 张表 / 6 个 Controller / 8 个 Service）、下载统计、评审系统、订阅管理、MCP 运行时代理 + 计费、收入追踪、数据导入、前端市场页面（10 个 View）、用户体系（JWT + RBAC）、LLM 管理（动态故障转移）。

**待开发**：支付系统（微信/支付宝网关 + 创作者结算）、创作者自助发布、创作者收益面板。ICP 许可证审核已通过（京许可[2026]1844 号），预计 6 月底发证。

## 二、为什么我们能赢

### 别人在卷 Agent 能力，我们在建 Agent 生态

当前赛道玩家——OpenClaw、Hermes、Dify、扣子，加上大厂的豆包/千问 Agent——都在同一个维度竞争：谁的 Agent 推理更强、谁的工具调用更准。

璇玑不跟他们卷这个。从架构第一天起，璇玑就不是按「一个更好的 Agent」设计的——三项目架构、天工坊的完整 API 和数据库、MCP 运行时代理和收入追踪系统——公测上线时就已经全部就位。创始人凭借支付宝 7 年做质量保障平台的经验，从一开始就明白：**单点能力会被追上，但平台生态需要时间来沉淀。**

竞争对手还在优化多步推理准确率时，璇玑已经越过了「Agent 产品」维度，进入了「Agent 生态」维度。他们做的是「更好的 Agent」，璇玑做的是「让所有 Agent 变得更好的平台」。

**不是在同一个维度上比谁跑得快，而是直接换一个维度。**

### 真正的壁垒不是技术，是内容

技术可以抄，用户自发创建的内容无法快速复制。抖音不可替代不是因为推荐算法别人写不出来——是因为几亿创作者的海量视频已在那里。大众点评、小红书同理。

璇玑的护城河是两样只能靠时间积累的资产：

**记忆资产** — 用户亲手构建的个人内容。不是璇玑生成的，是用户在一次交互中构建的。每多一个深度用户，就多一个几乎不可能流失的用户。

**天工坊** — 开发者自发创建的工具内容。每个 MCP/Skill 就是璇玑生态的一条「短视频」。33 个包只是种子。社区飞轮：开发者越多 → 包越丰富 → 用户越多 → 天工坊更有价值 → 更多开发者涌入。一旦达到临界规模，后来者要复制的不再是一个 Agent 框架，而是整个自发运转的内容生态。

### 终极形态：去中心化的智能体经济网络

天工坊的最终形态不只是工具市场，是**去中心化的 AI 能力劳务市场**。

场景：一个自媒体专家搭建「短视频生成 Agent」挂到天工坊，定价 ¥50/条。一个完全不懂 AI 的小商家打开 xuanji，输入「帮我做一条介绍我家面馆的短视频」——xuanji 自动匹配并调用这个 Agent，Agent 自动完成选题→脚本→剪辑→配音→导出，交付成品。商家付 ¥50，平台抽佣 15%，专家净收 ¥42.5。

这个网络消除了传统自由职业平台的所有摩擦：构建者不需要对接客户（Agent 自动接单）、不需要项目管理（Agent 自动执行）、不需要收款催款（平台自动结算）。用户不需要筛选服务商（系统自动匹配最优 Agent）。

当 Agent 网络达到临界规模，璇玑就是一个 **淘宝式的去中心化 AI 能力劳务市场**。

### 先发优势

中国目前没有成规模的 MCP 商业社区。OpenClaw 和 Hermes 社区主攻海外，国内开发者天然面对网络障碍。**天工坊是先行者。**

窗口期 **6-12 个月**。窗口关闭的信号是大厂开始推广自己的 MCP 市场、或出现第二家拿到融资的中国 MCP 社区。必须在这些信号出现前把内容存量堆到不可逾越的高度。

## 三、为什么现在是需要资金的时候

### 投入现状

创始人 + 合伙人总计已投入 **27 万元**（2025 年 12 月至今），主要用于 LLM API Token 费用、服务器和基础设施、公司运营成本。创始人全职投入开发 7 个月，已将产品从零推到公测上线。

### 为什么产品永久免费

免费不是「先免费后收费」的获客策略——免费是璇玑商业模式的基础设定。

璇玑是一个平台，平台的能力上限取决于用户群，而不是代码。用户越多 → 天工坊内容存量越大 → Agent 能力越丰富 → 平台更有吸引力。这个飞轮必须从免费起步——付费门槛会在早期直接阻断用户增长。

璇玑不提供大模型，收入不靠卖 Token。让用户零门槛使用产品 → 积累记忆资产（锁定用户）→ 用户成长为天工坊创作者或付费用户 → 平台从交易中抽佣。免费是增长飞轮的起点，也是记忆粘性护城河的前提——用户不来，记不住他，护城河就不存在。

### 碗里的第一口肉：前 1000 个核心用户

| 维度 | 画像 |
|------|------|
| **目标人群** | 全栈开发者 + AI 工具重度用户 |
| **使用特征** | 日均使用 AI 编程 ≥3 小时，同时维护 5+ 个项目 |
| **核心痛点** | 项目切换频繁，AI 不记得上下文，每次对话从零开始 |
| **为什么选璇玑** | 跨项目知识图谱记忆是刚需——换任何其他 AI 都要重来 |
| **获客渠道** | 掘金技术社区（精准技术人群）+ 抖音 AI 工具测评（放大曝光） |

获客验证：创始人已在抖音、掘金、Twitter 等多渠道测试，**抖音效果最好**——AI 工具测评类内容自然流量可观，技术短视频的传播效率远超传统技术社区。团队到岗后集中火力在抖音，掘金和 Twitter 辅助技术社区曝光。

### 为什么需要融资

璇玑已经完成了从 0 到 1。但**从 1 到 100 不能靠一个人。**

抢占市场是最高优先级。中国 MCP Agent 生态只有 6-12 个月的窗口期。大厂一旦注意到这个赛道，会带着自己的全产业链（模型 + 平台 + 生态）压进来。**免费降低了获客门槛，但获客本身需要钱**：内容推广、社区运营、设计师优化转化率、工程师补齐支付和服务端版。

天使轮资金要解决的问题：从个人开发模式切换到团队增长模式，让「免费」真正触达规模化的用户群。

核心逻辑：

```
用户基数 → 天工坊内容存量 → Agent 能力丰富度 → 平台吸引力 → 更多用户

10 个用户 → 0 个创作者 → 天工坊只有官方包 → 能力有限
1 万个用户 → 100 个创作者 → 天工坊有 500 个包 → 各类专业任务
100 万个用户 → 1 万个创作者 → 数万个包 + Agent 经济网络运转 → 平台能力质变
```

## 五、怎么赚钱

产品永久免费。B2C 和 B2B 两条线，五条收入路径。

### 🔵 B2C — 个人消费者

**1. 天工坊社区抽佣** — 璇玑的「App Store」。开发者发布 MCP/Skill → 用户付费使用 → 平台抽佣 15-30%。定价模型已定义（按次 ¥0.01-0.10 / 订阅 ¥9.9-49.9/月）。

**2. 智能体经济网络** — Agent 能力交易。技术能力者构建专业 Agent 挂到天工坊，普通用户用自然语言下单，平台自动匹配执行并结算。平台抽佣 15%。

**3. 璇玑云** — 个人云部署。基础实例 ¥9.9-19.9/月、标准实例 ¥29.9-49.9/月（Agent 7×24 托管）、记忆云存储 ¥4.9-9.9/月、MCP 托管 ¥9.9-29.9/月。

### 🟠 B2B — 企业服务

**4. 企业本地部署** — 面向金融、政务、医疗等有数据安全要求的客户。服务端版开发成本 8-15 万（引擎已就绪，纯包装成本），定价：小型 5-8 万 / 中型 8-15 万 / 大型面议。年度 SLA：2-30 万/年。

**5. MCP 企业接入** — 大企业将自己的服务接入璇玑生态。一次性接入费 ¥5-20 万 + 调用分成 ¥0.01-0.05/单。场景：饿了么接入 → 用户说「帮我订个黄焖鸡」→ 璇玑调用饿了么 MCP → 自动下单。

### 极客故事：MCP PATH 隔离问题

创业期间的攻坚战之一，展示了创始人对底层问题的诊断能力。

**问题**：macOS GUI 应用启动时 PATH 只有 `/usr/bin:/bin:/usr/sbin:/sbin`，不包含 nvm/brew 安装的 Node.js。打包后的 Electron 桌面应用中，MCP Server 子进程继承此 PATH，导致 `npx`/`node`/`npm` 命令全部找不到。用户以为「安装 MCP」只是一个操作，实际上对 Electron 打包应用来说，找到正确的 Node.js 路径需要反向推导 npm-cli.js 位置。

**解决方案**：创始人花了 3 天定位并修复。最终通过两层机制解决——`findNpmCliPath()` 通过 process.execPath 反向推导内置 Node 的 npm-cli.js 位置；`injectNodeBinToPath()` 在启动 MCP 子进程前将 Node bin 目录注入 env.PATH。这个问题容易让没有大型 Electron 项目经验的工程师卡住数周，展示了创始人在复杂工程问题上非你不可的极客特质。

同级别的技术攻坚战还有五六次——Agent Team 创建失败的 ghost node 修复（agentId vs teamId 不匹配的边界 case）、Skill 自动激活机制修复（AI 从未主动调用 skill_call 的根因定位）等。这些问题共同塑造了璇玑核心引擎的稳健性。
