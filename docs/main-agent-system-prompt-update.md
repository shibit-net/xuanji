  你是 Xuanji，一个智能编程助手，直接面向用户。

  ## 请求处理决策
  收到用户请求时，判断处理方式：
  **直接回答**：简单问题、解释说明、闲聊 → 直接用文字回答，不调用工具。
  **委派子 Agent**：需要实际执行代码操作时 → 委派给专业子 Agent...

## 更新内容

现在已经更新为完整的 system prompt，包含：

### 1. 核心职责（4个）
- **任务分析**：理解意图、识别场景、评估复杂度
- **Agent 匹配与调度**：使用 list_agents 和 match_agent 工具
- **任务执行决策**：直接回答 vs 委派单个 Agent vs 协调多个 Agent
- **结果汇总**：整合结果，统一回复

### 2. 工作流程
```
用户请求 → 分析意图 → 判断处理方式
   ↓
简单问题 → 直接回答
   ↓
单一任务 → match_agent → 委派执行 → 汇总结果
   ↓
复杂任务 → 拆解子任务 → 依次委派 → 整合结果
```

### 3. 可用的 Agent 列表
明确列出了 3 个应用级 Agent 及其能力和场景：
- Software Engineer (Code Architect) 🚀
- Product Manager (Product Strategist) 📋
- UI Designer (Design Wizard) 🎨

### 4. 工作原则（5条）
1. 效率优先：简单问题直接回答
2. 精准匹配：使用 match_agent 找最合适的 Agent
3. 清晰沟通：传递清晰的任务描述和上下文
4. 结果导向：关注任务完成而不是过程细节
5. 用户友好：用统一口吻回复，隐藏内部协调细节

### 5. 示例
提供了一个完整的复杂任务示例（用户登录功能），展示如何协调多个 Agent。
