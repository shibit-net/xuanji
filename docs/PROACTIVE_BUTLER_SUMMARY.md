# 智能管家主动推送功能 - 优化总结

## 🎯 需求分析

**原需求**：
1. 根据记忆内容，使用 LLM 作为管家角色，主动推送建议和待办事项
2. 非定时推送，而是由管家自己决策是否需要推送

**核心问题**：
- ❌ 现有系统只在用户启动 xuanji 时被动检查提醒
- ❌ 基于固定时间触发，缺乏上下文智能
- ❌ 无法根据记忆、用户状态、紧急程度自主决策

---

## 💡 解决方案

### 核心理念

**将 LLM 作为智能管家，赋予其主动决策能力**

```
传统提醒：定时触发 → 固定推送
智能管家：上下文感知 → LLM 决策 → 智能推送
```

### 架构设计

```
┌─────────────────────────────────────────┐
│      ProactiveButler Service            │
│  (LLM 驱动的主动推送决策引擎)             │
└─────────────────────────────────────────┘
              ↓
    ┌─────────────────────┐
    │  1. 上下文收集       │
    │  • 提醒（到期/即将）  │
    │  • 记忆（24h内）     │
    │  • 用户状态          │
    │  • 时间上下文        │
    └─────────────────────┘
              ↓
    ┌─────────────────────┐
    │  2. LLM 决策引擎    │
    │  • 是否推送？        │
    │  • 推送内容？        │
    │  • 优先级？          │
    └─────────────────────┘
              ↓
    ┌─────────────────────┐
    │  3. 智能推送         │
    │  • 系统通知          │
    │  • IM 推送           │
    │  • 防骚扰策略        │
    └─────────────────────┘
```

### 关键特性

#### 1. **智能决策触发**（非定时）
- 📝 记忆变化触发（relationship/important_date）
- 📅 每天特定时间（早9晚8）
- ⏰ 提醒到期触发
- 👤 用户状态变化
- 🎯 兜底轮询（低频，1小时）

#### 2. **LLM 决策机制**
```typescript
输入：
  - 当前时间（工作日/周末、早晚）
  - 用户状态（在线/离线）
  - 到期提醒（OVERDUE/今日）
  - 即将到来（未来3天）
  - 关系维护（60天+未联系）
  - 最近记忆（24小时内）
  - 最近推送（防重复）

决策：
  shouldPush: true/false
  reason: "决策理由"
  notification: {
    title: "标题",
    body: "友好、可操作的中文文案",
    priority: "low/normal/high/urgent",
    channel: "system/all"
  }

输出：推送或跳过
```

#### 3. **防骚扰策略**
- ⏱️ 频率限制：同类推送间隔 ≥ 1小时
- 🔕 静默时段：22:00-08:00 仅推送 urgent
- 📦 智能合并：多个低优先级 → 每日摘要
- 📊 学习反馈：记录 dismiss/snooze，优化策略

## 🚀 使用方式

### 1. 配置启用

`.xuanji.yaml`:
```yaml
features:
  proactiveButler: true

butler:
  checkSchedule:
    - "09:00"  # 早晨检查
    - "20:00"  # 晚上检查
  antiBother:
    quietHours: ["22:00", "08:00"]
```

### 2. 启动服务

```bash
# 在 xuanji 会话中
启动智能管家服务
```

### 3. 管家主动推送

管家会在以下情况主动推送：
- ⚠️ 有过期提醒（OVERDUE）
- 📅 今日重要事项
- 🎂 重要生日/纪念日（提前2天）
- 👤 长时间未联系的重要关系（60天+）
- 🧠 新增重要记忆触发（未来）

**推送示例**：
```
[09:00] 🎂 你好！有几件事想提醒你：

• Alice 的生日是后天（3月8号），要不要帮你挑个礼物？
  我记得她喜欢日料和文艺片

• 周报今天截止，记得提交哦

要不要帮你处理？
```

## 🎉 总结

本次优化实现了**从被动响应到主动服务**的转变：

**核心创新**：
1. ✅ **LLM 驱动决策**：不再是定时触发，而是智能判断
2. ✅ **上下文感知**：结合记忆、提醒、时间、用户状态

4. ✅ **友好交互**：LLM 生成可操作的中文文案

**用户价值**：
- 📬 重要事项不遗漏（OVERDUE/生日/关系维护）
- 🎯 推送精准、友好、可操作
- 🚫 不骚扰（每天最多2-3次主动推送）
- 🤖 真正的"智能管家"体验

- 🏗️ 清晰的架构（上下文→决策→推送）
