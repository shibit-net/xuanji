# 分层提示词系统

璇玑采用 L0-L2 三层提示词架构，既有统一的基础规则，又能灵活适应不同场景。

---

## 架构概览

```
L2: 协调层 - 团队协作、复杂任务协调
L1: 场景层 - 10+ 专业场景（代码、调试、设计等）
L0: 基础层 - 通用规则、安全、工作流
```

---

## 各层详解

### L0: 基础层

**职责**：定义 Agent 的基本身份和行为准则。

**内容包括**：
- Agent 身份定位
- 安全和伦理规范
- 基本工作流程
- 记忆系统使用指南
- 架构思维和推理能力

**特点**：
- 所有 Agent 共享
- 保持一致性和可靠性
- 不会频繁变动

---

### L1: 场景层

**职责**：针对特定专业领域提供专门的提示词。

**内置场景**：
| 场景名称 | 用途 |
|---------|------|
| write_code | 编写代码 |
| debug | 调试和排错 |
| refactor | 代码重构 |
| review | 代码审查 |
| design | UI/UX 设计 |
| deploy | 部署和运维 |
| research | 研究和资料整理 |
| learn | 教学和学习指导 |
| finance | 金融和投资分析 |
| general | 通用任务 |

**特点**：
- 每个场景都有专业提示词
- 可根据需要扩展新场景
- 场景切换快速流畅

---

### L2: 协调层

**职责**：处理多智能体协作和复杂任务。

**内容包括**：
- 团队协作规则
- 任务分配策略
- 结果汇总方法
- 冲突解决机制

**特点**：
- 仅在多 Agent 场景使用
- 与协作策略配合使用
- 灵活应对复杂场景

---

## 动态加载机制

### 工作流程

```
用户输入
    ↓
意图分析器 (Scene Classifier)
    ↓
识别场景和复杂度
    ↓
动态加载对应提示词
    ↓
组合 L0 + L1 (+L2)
    ↓
生成高质量响应
```

### 优势

1. **节省 Token**：只加载需要的提示词
2. **更专业**：场景专用提示词更精准
3. **可扩展**：添加新场景不需要改核心代码
4. **自适应**：根据对话上下文智能切换

---

## 自定义场景

### 如何添加新场景

1. 创建新的场景提示词文件
2. 在场景分类器中添加识别规则
3. 测试和调整

### 场景提示词模板

```
# [场景名称]
## 目标
[这个场景要完成什么]

## 最佳实践
[专业建议和技巧]

## 示例
[具体示例]

## 注意事项
[需要注意的地方]
```

## 与记忆系统的配合

### 个性化微调

分层提示词系统与记忆系统紧密配合：
- 基础提示词保持一致
- 根据用户画像微调整体风格
- 场景提示词根据用户偏好动态调整

### 持续优化

- 用户反馈 → 更新记忆 → 影响提示词选择和微调
- 形成闭环，越用越懂你
