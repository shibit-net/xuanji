三个月前我开始做一个东西，现在觉得可以拿出来聊聊了。

## 为什么要做

市面上 AI 助手很多，但我发现它们都有三个共同的毛病：

**1. 记不住关系。** 你说「Morgan 不吃辣」「Pepper 上次说那家意大利菜不错」，它们能存下这些句子。但下次你说「帮我家订个餐厅」，它们不知道 Morgan 是你女儿、Pepper 是你妻子、那家餐厅 Pepper 去过。它们存的是文档，不是知识。

**2. 只会单打独斗。** 你让它「调取测试数据、对比三种方案、跑模拟、出报告」，它只能一步步来。OpenClaw 是一个子 Agent 干到底，Hermes Agent 是几个人各干各的——没有分工，没有配合，更没有辩论。三个人意见不同，你得自己当裁判。

**3. 不会改。** 你说它错了，当时改了，下次又犯。因为没有反馈回路把纠正写回记忆。

还有一件事让我受不了——很多框架把几百个工具定义一次性灌进 system prompt，LLM 的注意力是有限的，相当于考试时在桌上摊一百本无关的书。

## 璇玑做了什么不一样的事

github.com/shibit-net/xuanji

- **知识图谱记忆**：不是存文档，是建实体+关系+事件的图。Morgan 是一个实体，连着「不吃辣」「爱吃芝士汉堡」「是 Tony 的女儿」。信息不是存着，而是连着。

- **5 种协作策略**：串行（流水线）、并行、层级（Leader 分配）、辩论（专家争论达成共识）、流水线（数据流转）。没有合适的 Agent？当场创建一个。

- **用户纠正 → 永久修正**：你纠正一次，图谱更新，下次自动避开。不是靠 LLM 猜什么重要。

- **L0-L2 三层 Prompt**：基础层始终加载、场景层只加载 1-3 个、协调层只在复杂任务时出现。不需要的规则不占注意力。

- **MCP + Skills 按需加载**：用不到的不注入 prompt。需要浏览器？加载 Playwright MCP。需要操控桌面？加载 computer-use MCP。

- **数据完全本地**：所有对话、记忆、API Key 加密存储在本地，零遥测。

## 技术栈

TypeScript + Electron + React + SQLite + Cytoscape 图谱可视化。AGPL-3.0 开源。

## 现在的状态

诚实地说，还有很多不足——模型推理天花板、复杂场景偶发不稳定、图谱准确度随数据量衰减。但我们在持续迭代，每周都有新版本。
