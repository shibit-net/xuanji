# Agent Team 快速开始

## 什么是 Agent Team？

Agent Team 允许你创建一个由多个专业化 AI agent 组成的团队来协同完成复杂任务。每个 agent 有自己的角色和专长，团队通过不同的策略协作。

## 5 秒快速入门

只需告诉 Xuanji：

```
"帮我用团队模式审查 src/core/agent/team/TeamManager.ts 这个文件，
需要架构师检查设计、安全专家检查漏洞、性能分析师优化性能"
```

Xuanji 会自动调用 `agent_team` 工具创建一个代码审查团队。

## 核心概念

### 团队成员角色

- `explore`: 探索型（只读，用于搜索、分析）
- `plan`: 规划型（只读，用于设计、架构）
- `coder`: 编码型（可读写，用于编写代码）
- `general-purpose`: 通用型（全能）

### 协作策略

1. **sequential（串行）** - 按顺序执行，每个成员看到前面的结果
2. **parallel（并行）** - 同时工作，独立输出
3. **hierarchical（层级）** - 主 agent 协调，其他成员执行
4. **debate（辩论）** - 多轮讨论达成共识
5. **pipeline（流水线）** - 前一个的输出是下一个的输入

## 使用场景

### 场景 1: 代码审查（串行）

```
"创建代码审查团队分析新功能的代码质量"
```

**适合**：需要多角度审查（架构 → 安全 → 性能）

### 场景 2: 信息搜集（并行）

```
"用并行团队从文档、开源项目、博客三个维度研究微服务最佳实践"
```

**适合**：可独立进行的搜索任务

### 场景 3: 架构设计（辩论）

```
"让架构师团队辩论离线优先应用的同步策略，考虑性能、简洁性和可扩展性"
```

**适合**：需要权衡多种方案的决策

### 场景 4: 数据处理（流水线）

```
"创建数据流水线：提取 GitHub issues → 清洗去重 → 分类分析 → 生成洞察报告"
```

**适合**：有明确的数据加工流程

### 场景 5: 特性开发（层级）

```
"用层级团队开发团队性能分析功能：技术负责人设计，后端/前端/测试分工实现"
```

**适合**：需要总体规划和分工协作

## 参数说明

当你让 AI 使用团队时，它会配置以下参数：

- `team_name`: 团队名称
- `goal`: 团队目标
- `strategy`: 协作策略
- `members`: 成员列表
  - `id`: 成员标识
  - `role`: 角色类型
  - `capabilities`: 能力描述
  - `priority`: 优先级（决定执行顺序）
  - `system_prompt`: 成员特定指令

## 最佳实践

✅ **DO**:
- 选择 3-5 个成员（太多会增加成本和复杂度）
- 明确每个成员的职责和能力
- 根据任务性质选择合适的策略
- 设置合理的 timeout 和 max_rounds

❌ **DON'T**:
- 为简单任务创建团队（浪费资源）
- 成员能力重叠过多（缺乏分工）
- 在 parallel 模式下期望成员间共享信息
- 忘记设置 max_rounds（debate 可能无限循环）

## 性能考虑

- **Token 消耗**: 团队 = 所有成员消耗之和
- **并行策略**: 虽然快，但消耗最多（所有成员同时工作）
- **辩论策略**: 可能需要多轮，注意设置 max_rounds
- **超时时间**: 默认 10 分钟，复杂任务可能需要更长

直接用自然语言描述你的需求，Xuanji 会自动判断是否需要使用团队模式，并配置合适的成员和策略。

例如：
- "用团队模式..."
- "创建一个包含...的团队"
- "让多个 agent 协作..."

- "逐步分析..."（会自动选 sequential 或 pipeline）
