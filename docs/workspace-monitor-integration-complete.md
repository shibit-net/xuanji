## 显示内容

### 主流程面板（顶部）

显示 Xuanji 主 agent 的内部执行阶段：

1. **意图分析** - 识别场景和复杂度
2. **任务规划** - 选择 Agent 和策略
3. **任务执行** - 调用子 Agent 执行
4. **结果汇总** - 合并多个结果（可选）

每个阶段显示：
- 状态（pending / running / completed / error）
- 耗时
- 详细数据（点击展开）

### Canvas 区域（中间）

显示 Agent 树形结构：
- Xuanji (主 Agent)
- 子 Agent (coder / explore / plan 等)
- 工具调用
- 实时状态

### 统计面板（底部）

显示整体统计：
- 累计 Token
- 当前调用 Token
- 迭代次数

## 设计特点

### 1. 紧凑布局

- 使用更小的字体和间距
- 横向滚动支持多个阶段
- 最大高度限制，避免占用过多空间

### 2. 状态指示

- **pending**: 灰色，半透明
- **running**: 蓝色，带边框高亮
- **completed**: 绿色
- **error**: 红色

### 3. 实时更新

- 通过 `workspaceStore` 订阅状态变化
- 自动更新 UI
- 显示当前正在执行的阶段

### 4. 详细信息

点击阶段可查看详情：
- 意图分析：场景、复杂度、匹配方式、置信度
- 任务规划：策略、Agent 选择、任务列表
- 任务执行：成功/失败、耗时、错误信息

## 数据流

```
MainAgent 执行
  ↓
发射 Hook 事件
  ↓
agent-bridge.ts 监听
  ↓
IPC 发送到 Renderer
  ↓
workspaceStore 接收
  ↓
更新阶段状态
  ↓
MainFlowVisualization 重新渲染
```
