## 修复 #1：API 上下文破坏问题

### 问题描述

工具执行后，如果 API 返回 429（频率超限）或其他错误，会破坏消息上下文：
- 连续两条 user 消息（tool_result + 新用户输入）
- API 代理（shibit-llm）报错
- 用户重试时触发更多错误，形成恶性循环

### 解决方案

**1. 消息历史回滚机制**
```typescript
// 工具执行前保存快照
const messageSnapshot = this.messageManager.saveSnapshot();

// API 失败时回滚
if (!streamResult.result) {
  this.messageManager.restoreSnapshot(messageSnapshot);
  throw error;
}
```

**2. 429 错误自动冷却重试**
```typescript
// 429 错误 → 冷却 60 秒 → 自动重试 1 次
if (isRateLimitError(error) && retryCount === 0) {
  await sleep(60_000);
  return this.executeWithRetry(..., retryCount + 1);
}
```

**修改前**：

## 修复 #2：输出后空白问题

### 问题描述

Xuanji 每次输出完内容后，会流出来半个屏幕的空白：
- 流式文本区域在 idle 时仍然渲染
- 用户需要滚动才能看到输入框
- 影响终端使用体验

### 解决方案

**1. 优化流式文本渲染条件**
```tsx
// 添加 status !== 'idle' 条件，idle 时隐藏流式文本区域
{renderedStreamLines && status !== 'idle' && !pendingPermission && !pendingPlanReview && !pendingUserQuestion && (
  <Box marginLeft={2} flexDirection="column">
    {renderedStreamLines.map((line, i) => (
      <Box key={i}><Text>{line}</Text></Box>
    ))}
  </Box>
)}
```

**2. 立即归档流式文本**
```typescript
onEnd: (state: AgentState) => {
  // ★ 立即归档流式文本到 Static ★
  archiveStreamText();

  // 清空动态区域
  streamTextRef.current = '';
  streamBufferedRef.current = false;
  setStreamText('');
  setStreamProgress(0);
}
```

## 用户体验改进

### 稳定性

- ✅ **API 错误自动恢复**：429 错误后自动冷却 60 秒并重试
- ✅ **上下文保护**：API 失败时回滚消息历史，避免结构破坏
- ✅ **错误隔离**：单次 API 失败不影响后续对话

- ✅ **更快的响应**：减少动态区域渲染，Ink 重绘开销降低

- ✅ **更清晰的视觉层次**：静态历史 + 动态输出 + 独立统计

### 可靠性

- ✅ **消息历史一致性**：快照回滚机制确保上下文完整
- ✅ **API 兼容性**：与 Anthropic/OpenAI 官方 API 和 shibit-llm 代理兼容
- ✅ **测试覆盖**：5 个新测试 + 1166 个回归测试通过
