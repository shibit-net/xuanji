# 掘金推广文案

**标题**：我一个人写了三个月，开源了一个多智能体 AI 管家——璇玑 (Xuanji)

**封面建议**：架构 Mermaid 图或知识图谱截图

---

## 它不是聊天机器人，而是一个住在你电脑里的 AI 管家

看过《钢铁侠》的都知道贾维斯。Tony Stark 早上醒来，贾维斯已经在汇报天气、日程、战甲状态。Tony 说「帮我把 Mark 42 的推进系统优化一下」，贾维斯开始干活——调取历史测试数据、对比不同合金的耐热曲线、跑 CFD 模拟、出对比报告。

Tony 从来不需要向贾维斯解释「我说的是上周那种合金」。不需要每次对话都重新介绍自己。**贾维斯有三个东西，任何 AI 助手要做到它那样，缺一不可：**

- 它认识你。不是「存下了你说过的话」，而是真的知道你是谁。
- 它不只是回答，它能做事。一个任务不管涉及多少种工具，从头到尾自己串下来。
- 它会学。你说它做错了，它改。而且下次不会再犯。

**这，就是璇玑想成为的样子。**

---

## 现在 AI 助手的三个通病

### 通病一：记不住关系

你说「Morgan 不吃辣，爱吃芝士汉堡」「Pepper 上次说那家意大利菜不错」，再问「帮我家订个餐厅」——绝大多数助手只有向量搜索，它搜到了这些句子，但不知道 Morgan 是女儿、Pepper 是妻子、那家餐厅 Pepper 去过。

璇玑会把信息拆成实体、关系和事件，建一张可推理的图。下次你问，它沿着关系网走：家庭成员 → 各自偏好 → 综合推荐。

### 通病二：只会单打独斗

复杂的工程任务需要分工协作。OpenClaw 只能一个子 Agent 干到底；Hermes Agent 可以并行几个 Agent，但没有辩论机制——意见不同时你得自己当裁判。

璇玑有 5 种协作策略：串行流水线、并行分析、层级分派、专家辩论、数据流转。方案难分高下时切辩论模式，两个专家各自辩护。没有合适的 Agent？当场创建一个专用的。

### 通病三：反复犯错不改

你说「上次那个方案不行」，它当时改了，下次又推荐。因为没有反馈回路把纠正写回记忆。

璇玑的反馈循环：你纠正一次 → 图谱永久更新 → 下次自动避开 → 你不需要再说一遍。

## 璇玑做了什么不一样的事

### 🧠 知识图谱记忆——信息有线连着
实体-关系-事件模型，Cytoscape 可视化，用户反馈驱动图谱更新。

### 🚀 5 种多智能体协作策略
串行 | 并行 | 层级 | 辩论 | 流水线。不是一个人干活，也不是几个人各自干活——是一支有分工、有配合、可以当场扩编的团队。

### 📚 L0-L2 动态 Prompt
基础层始终在线，场景层按需加载 1-3 个，协调层只复杂任务触发。不加载多余上下文到 LLM 注意力窗口。

### 🔌 用到才加载的 MCP + Skills
不把所有工具定义灌进 system prompt。需要浏览器？加载 Playwright MCP。天工坊一键安装更多。

### 🔒 数据完全本地
对话、记忆、API Key 全部加密存本地，零遥测。
