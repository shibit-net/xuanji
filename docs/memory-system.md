# 记忆驱动学习系统

璇玑的记忆系统不仅仅是简单存储，更是主动学习和成长的核心。

---

## 核心特性

### 1. 实体-关系-事件模型

**什么是实体-关系-事件？**

- **实体 (Entity)**：存储的信息片段，如人、事、物
- **关系 (Relation)**：实体之间的连接
- **事件 (Event)**：一段时间内发生的活动

**示例**：
```
实体: "React"
关系: "擅长"
实体: "前端开发"
事件: "2026-06-08 学习 React 组件"
```

---

### 2. 知识图谱可视化

**Cytoscape 知识图谱**：
- 直观展示实体和关系
- 支持交互式探索
- 点击节点查看详情
- 关系强度可视化

---

### 3. 语义搜索

**如何工作？**：
- 不仅仅是关键词匹配
- 理解上下文和语义
- 支持自然语言查询

**示例查询**：
```
"上次我们讨论的 React 性能优化方案是什么？"
```

---

### 4. 遗忘机制（艾宾浩斯遗忘曲线）

**为什么需要遗忘？**：
- 避免记忆无限膨胀
- 保留重要的信息
- 模拟人类学习过程

**工作原理**：
- 不常用的记忆逐渐衰退
- 重要的记忆通过强化被保留
- 定期清理过时信息

---

## 记忆驱动学习闭环

### 学习流程

```
用户反馈 → 事件记录 → 记忆更新 → 提示词调整 → 下次更好的表现
```

### 反馈信号类型

| 信号类型 | 描述 |
|---------|------|
| 明确肯定 | 用户明确说"好"、"对" |
| 明确否定 | 用户明确说"不对"、"改一下" |
| 积极互动 | 用户追问、深度讨论 |
| 消极互动 | 用户打断、跳过 |
| 修正行为 | 用户纠正璇玑的输出 |

### 自动优化机制

**每周后台任务**：
- 扫描上周的所有互动事件
- 分析反馈模式
- 调整提示词中的重点
- 更新用户画像
- 优化 Agent 行为

---

## 用户画像系统

### 画像维度

- **技术栈偏好**：喜欢用什么技术
- **沟通风格**：喜欢详细还是简洁
- **项目类型**：常做什么类型的项目
- **风险偏好**：保守还是激进
- **学习速度**：接收新概念的速度
- **安全意识**：对操作安全的要求

### 画像更新触发条件

- 积累一定数量的反馈后
- 每周例行优化
- 用户主动设置偏好

---

## 记忆存储架构

### 数据结构

```
Memory
├── EpisodicMemory（情景记忆）：完整对话记录
├── SemanticMemory（语义记忆）：提炼的知识
├── MemoryGraph（知识图谱）：实体关系网络
└── UserProfile（用户画像）：个性化信息
```

### 权限隔离

- 多用户支持
- 每个用户的记忆完全隔离
- 加密存储敏感信息

## 使用场景示例

### 场景 1：持续学习编程风格

**第一次**：
- 你要求写一个 React 组件
- 璇玑按默认风格写
- 你说"我更喜欢用 TypeScript，加上详细注释"

**记忆记录**：
- 实体：你
- 关系：偏好
- 实体：TypeScript、详细注释

**第二次**：
- 你又要求写组件
- 璇玑自动用 TypeScript + 详细注释
- 你说"完美！"

**强化学习**：
- 这个偏好被标记为高权重
- 以后会优先考虑

---

### 场景 2：记住项目上下文

**第一周**：
- 讨论项目架构选型
- 最终选择了 PostgreSQL 而不是 MongoDB
- 记忆系统记录了这次决策和原因

**第二周**：
- 讨论数据库优化
- 璇玑记得用的是 PostgreSQL
- 直接给出 PostgreSQL 相关的建议
