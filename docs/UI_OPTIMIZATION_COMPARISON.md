## 4. 国际化文件组织

### 优化前（单一文件）
```
src/core/i18n/
├── index.ts
└── messages.ts  (300+ 行，所有翻译混在一起)
```

```
src/core/i18n/
├── index.ts
├── locales/
│   ├── zh_common.ts      (70 行 - 通用词汇)
│   ├── en_common.ts      (70 行 - 通用词汇)

│   ├── en_settings.ts    (79 行 - 设置相关)

│   ├── en_session.ts     (待添加 - 会话相关)
```

**改进点**:
- ✅ 按功能分类（易于维护）
- ✅ 减少合并冲突（多人协作）
- ✅ 支持懒加载（性能优化）
- ✅ 更清晰的职责划分

## 5. 命令注册对比

### 优化前
```typescript
slashCommandRegistry.register({
  name: '/help',
  description: '显示帮助信息',
  handler: async () => {
    console.log(slashCommandRegistry.formatHelp());
  },
});
```

### 优化后
```typescript
slashCommandRegistry.register({
  name: '/help',
  description: '显示帮助信息',
  group: '基础',                    // 新增：分组
  icon: '📖',                        // 新增：图标
  usage: '/help [command]',          // 新增：用法
  aliases: ['/h', '/?'],             // 新增：别名
  hidden: false,                     // 新增：是否隐藏
  handler: async () => {
    setShowHelpPanel(true);          // 显示交互式面板
  },
});
```

**改进点**:
- ✅ 更丰富的元数据
- ✅ 支持别名和隐藏
- ✅ 使用示例说明
- ✅ 图标和分组支持

## 7. 技术债务清理

### 优化前的问题
- ❌ Help 信息硬编码在 messages.ts
- ❌ 命令列表无法动态生成
- ❌ 国际化文件臃肿难以维护
- ❌ Settings 界面单调
- ❌ 快捷操作无优先级控制

### 优化后的解决方案
- ✅ Help 从 SlashCommandRegistry 动态生成
- ✅ 支持按分组、图标、别名组织
- ✅ 国际化按功能模块拆分
- ✅ Settings 视觉效果提升
- ✅ 快捷操作支持优先级排序
